# 简介：Alist中的反射型跨站脚本攻击（XSS）漏洞 - CVE-2024-47067

Alist的某些版本中存在一个反射型跨站脚本（XSS）漏洞，可能允许未认证用户通过特制链接盗取其他用户的JWT令牌。这种情况下，攻击者可能会获取、删除或读取与该用户相关的文件。

## 技术背景

- **Alist版本**：3.28.0
- **漏洞类型**：反射型跨站脚本（XSS）
- **受影响的端点**：`/i/:link_name`
  - 该端点接受用户提供的值，并将其反射回响应中，导致XSS漏洞。
  - 返回的响应为`application/xml`，并且通过XHTML允许使用HTML标签。

## 漏洞影响

1. **文件操作**：攻击者可能会执行任意文件删除、读取或下载操作。
2. **JWT令牌泄露**：点击特制链接后，当前用户的JWT令牌会在弹窗中显示。

**点击此链接查看JWT令牌：**
```
http://alist.domain/i/JTY4...
```
> 注意：由于此问题的未认证特性，我们在此报告但不将其标记为漏洞。

## 安全风险

如果启用访客用户功能，攻击者可以在未认证的情况下代表访客用户发送请求。这是由于`Access-Control-Allow`头设置为所有来源（`*`）。

例如：
- 如果内部网络的Alist服务器被外部访问，通过访问特制网站，攻击者可以让已连接到该网络的其他访客用户执行任意请求。
- 假设访客用户可以删除特定目录的文件，攻击者可以引导其删除该目录。

### 漏洞测试示例

1. 创建一个`index.html`文件，内容如下（示例代码）：
   ```html
   <script>
       fetch("/api/fs/remove", { method: 'DELETE' });
   </script>
   ```
2. 当受害者访问此文件时，将发送请求删除`/tmp/test/test.txt`中的文件。
