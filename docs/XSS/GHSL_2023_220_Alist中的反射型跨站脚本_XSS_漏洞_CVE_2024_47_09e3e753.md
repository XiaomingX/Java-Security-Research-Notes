# GHSL-2023-220: Alist中的反射型跨站脚本(XSS)漏洞 - CVE-2024-47067

## 概述

在Alist中发现了一种反射型跨站脚本(XSS)漏洞，可能允许未认证用户通过特制链接窃取点击用户的JWT令牌。这可能导致未认证用户对连接服务或本地文件的任意复制、删除和读取。

## 技术背景

- **软件版本**: Alist 3.28.0
- **漏洞路径**: `/i/:link_name` 这个接口处理用户提供的值，并将其返回在响应中。由于返回的内容类型为`application/xml`，可以利用XHTML插入HTML标签，从而导致XSS漏洞。

## 漏洞影响

这一漏洞可能导致以下情况：
- 任意文件删除
- 任意文件读取
- 下载任意文件

## 漏洞证明

用户登录后，如果点击以下链接，其当前JWT令牌将通过弹窗显示：

```
http://alist.domain/i/JTY4...
```

**注意**: 由于此漏洞未认证的特性，我们虽报告了此问题，但不将其标记为严重漏洞。

## 攻击方式

如果启用了访客用户权限，未认证用户可以访问应用程序。攻击者可以通过设置`Access-Control-Allow`头，使其可以向应用程序发起请求。

例如，如果Alist服务器位于内部网络并假设外部不可访问，攻击者可以通过访问特定网站诱使内部网络的用户执行任意请求，如删除文件。

### 示例代码

创建一个`index.html`文件，内容如下：

```html
<!-- 示例HTML代码，执行API请求 -->
```

当受害者访问`index.html`时，会向`/api/fs/remove`发送请求，删除位于`/tmp/test/test.txt`的文件。如果访客用户有权限删除该文件，则该操作将执行。
