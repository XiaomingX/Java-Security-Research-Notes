# GHSL-2024-015: livemarks浏览器扩展中的跨站请求伪造（CSRF）漏洞 - CVE-2024-30252

## 背景
livemarks浏览器扩展存在一个跨站请求伪造（CSRF）漏洞。恶意网站可以诱使该扩展向任意URL发送经过身份验证的GET请求。

## 技术细节

- **那个版本**：v3.6
- **关键脚本**：`subscribe.js`使用当前 URL 中的第一个参数作为要订阅的RSS源的URL，并检查该RSS源是否为有效的XML。

### 攻击路径
- **网站访问**：由于`subscribe.js`被在`manifest.json`中声明为可访问的Web资源，攻击者可以通过其网页访问该脚本。
- **潜在后果**：这个问题可能导致权限提升。盲目的CSRF攻击会破坏运行在私有网络上的服务器的完整性。用户可能有一个被认为是安全的私有服务器，但真实情况可能并非如此，一旦收到攻击者发起的经过身份验证的请求，服务器的安全性就会被破坏。

### 漏洞利用条件
- 脆弱性需要UUID泄露。此扩展仅适用于Firefox。
- 如果攻击者能说服用户安装一个具有管理权限的额外扩展，尽管这个权限不允许直接进行身份验证请求，攻击者可能就能泄露UUID，从而利用CSRF进行进一步攻击。
