# GHSL-2023-145/GHSL-2023-151/GHSL-2023-165/GHSL-2023-172：stb_image和stb_vorbis的内存访问漏洞

## 概述
stb_image.h 和 stb_vorbis 库存在多个不同严重级别的内存访问漏洞。

### 受影响的版本
- stb_image：v2.28
- stb_vorbis：v1.22

## 漏洞详情

### 1. 内存越界读取
- **触发条件**：当设置 `stbi_set_flip_vertically_on_load` 为 TRUE，且 `req_comp` 的值与实际像素组件数量不匹配时。
- **漏洞利用**：攻击者可以利用特制的图像文件来泄露内部内存分配信息。

### 2. 未初始化的内存读取
- **触发条件**：`stbi__getn` 函数在读取字节时未检查返回值，导致使用未初始化的内存。
- **漏洞利用**：可能会导致信息泄露。

### 3. 双重释放内存
- **触发条件**：在 `stbi__load_gif_main` 中，可能尝试双重释放一个变量。
- **漏洞利用**：此漏洞可能仅在多线程环境中被利用，导致安全问题。

### 4. 空指针访问
- **触发条件**：`stbi__convert_format` 中 `src` 为 NULL。
- **漏洞利用**：此情况可能导致拒绝服务（DoS）。

### 5. 内存泄漏或双重释放
- **触发条件**：在失败情况下，未正确管理延迟内存。
- **漏洞利用**：可能导致内存泄漏或双重释放。

### 6. 越界写入
- **触发条件**：处理特制文件时，触发数组越界写入。
- **漏洞利用**：可能导致代码执行的风险。

### 7. 内存分配失败
- **触发条件**：内存分配函数返回异常，车辆未重置指针。
- **漏洞利用**：后续可能导致空指针解引用，引发拒绝服务。

### 8. 整数溢出
- **触发条件**：在内存分配过程中发生整数溢出。
- **漏洞利用**：攻击者可以控制分配的内存大小，导致潜在的代码执行。
