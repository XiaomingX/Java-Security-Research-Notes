# GHSL-2024-013_GHSL-2024-014: Meshery中的SQL注入漏洞 - CVE-2024-35181, CVE-2024-35182

在Meshery版本最高到v0.7.22中，存在SQL注入漏洞。这使得远程攻击者可以获取敏感信息、修改数据库记录，或通过两个HTTP端点的排序参数创建任意文件。

## 漏洞概述

- **项目**: Meshery
- **受影响版本**: 0.7.18
- **API接口**:
  - `/api/system/meshsync/resources/kinds`
    - 查询参数`order`直接用于构建SQL查询。
  - `/api/v2/events`
    - 查询参数`sort`同样直接用于构建SQL查询。

## 漏洞影响

- 攻击者可以:
  - 利用SQL注入执行任意SQL查询。
  - 写入任意文件。
  - 访问和修改数据库中的数据，例如性能配置、应用数据和Kubernetes配置。

### 漏洞利用

1. **文件写入**: 
   - 攻击者可以通过特定的SQL注入载荷写文件到任意位置。

2. **数据创建**: 
   - 可以创建任意数据库条目。

3. **信息提取**:
   - 使用盲注技术提取数据库信息，例如利用`sqlmap`自动化提取性能配置中的请求cookies。

## 漏洞发现

这些问题由GitHub的成员@atorralba（Tony Torralba）发现，使用CodeQL的SQL注入查询进行了检测。
