# GHSL-2023-081_GHSL-2023-082: Autolab中的Tar Slip漏洞 - CVE-2023-32676, CVE-2023-32317

## 概述

在Autolab中发现了两个Tar Slip漏洞，可能允许攻击者在文件系统中创建或替换文件，最糟的情况下会被应用程序或系统执行。

## 受影响的版本

- Autolab v2.10.0

## 漏洞详情

### 1. **安装评估功能中的漏洞**
- **攻击者要求**：需具备身份验证的讲师权限。
- **漏洞利用方式**：攻击者上传一个特制的Tar文件。
- **漏洞机制**：上传的Tar文件可以包含指向目标目录外部的文件路径（例如：`../../../../tmp/tarslipped1.sh`）。当表单提交时，Tar文件中的文件会被解压到攻击者选择的位置。

   示例代码片段：
   ```python
   # 此代码将文件写入攻击者定义的目标位置
   relative_pathname = full_name_from_archive
   ```

- **注意事项**：一旦成功利用，`/tmp`文件夹中会出现名为`tarslipped1.sh`的文件。

### 2. **MOSS作弊检测功能中的漏洞**
- **攻击者要求**：同样需具备身份验证的讲师权限。
- **漏洞利用方式**：攻击者可以上传特制的Tar文件到“Base File Tar”或“Additional File Archive”。
- **漏洞机制**：在MOSS作弊检测期间，Tar文件中的文件可以被解压到攻击者选择的位置。

   示例代码片段：
   ```python
   # 此代码将文件写入攻击者定义的目标位置
   destination = pathname_from_archive
   ```

- **注意事项**：成功利用后，`/tmp`文件夹中会出现名为`tarslipped2.sh`的文件。

## 潜在危险

这些漏洞可能导致：
- 任意文件写入。
- 远程代码执行（如果可以写入并执行文件）。

## 漏洞复现

要重现这些漏洞，攻击者可以创建特制的tar档案并遵循上述步骤进行上传。
