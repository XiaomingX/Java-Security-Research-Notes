# GHSL-2023-028: Jellyfin中的远程代码执行漏洞 - CVE-2023-48702

## 概述

在Jellyfin服务器上，具备管理员权限的用户可以通过特定的接口执行任意代码。

## 技术细节

- **漏洞位置**：`/System/MediaEncoder/Path`接口
- **漏洞类型**：远程代码执行（RCE）

### 漏洞原理

1. **执行任意文件**：该接口可以通过`ProcessStartInfo`函数运行任意执行文件。
2. **恶意管理员行为**：
   - 恶意管理员可以设置一个网络共享，并提供一个指向该共享下可执行文件的UNC路径。
   - 当Jellyfin调用`/System/MediaEncoder/Path`时，它将执行这个可执行文件。

### 发现方法

这个漏洞是通过CodeQL的C#查询发现的。

### 漏洞利用示例

- **上传可执行文件**：管理员可以使用`/Branding/Splashscreen`接口，因该接口未能对文件类型进行充分验证：
  - 发送请求时指定了合适的内容类型（如image/png）。
  - 实际上传的是一个可执行文件。
  
- **结合漏洞进行远程代码执行**：
  - 由于`ProcessStartInfo`会忽略文件扩展名，导致上传的可执行文件得以运行。

## 潜在后果

此漏洞可能导致在身份验证成功后，攻击者远程执行代码。
