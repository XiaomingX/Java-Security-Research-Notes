# GHSL-2020-228: YApi中的弱JSON Web令牌（JWT）签名秘密 - CVE-2021-27884

## 问题概述

YApi中使用了弱随机数生成器来签署JSON Web令牌（JWT），这可能导致安全漏洞。

## 技术细节

- **漏洞简介**  
  使用`Math.random`生成的随机字符串用作JWT的签名密钥。

- **用户创建**  
  当新用户创建时，`randStr`函数生成一个密码盐（passsalt），用于：
  - 哈希密码
  - 签名JWT

- **生成机制**  
  - `Math.random`返回0到1之间的浮点数。
  - 通过`toString(36)`将数字转换为不同的字符。  
    - 例如：`0.19280841320093556`被转为`0.6xvo3g36129`，去掉前两个字符后，结果为`6xvo3g36129`。
  - 生成的密钥通常为10到12个字符，由数字和小写字母组成。  
  - 去掉的部分总是`0`，这使得计算变得可逆。

- **弱点分析**  
  - 不安全的伪随机数生成器可以被攻击者利用。
  - 通过观察三个连续生成的值，攻击者可以重建生成器的内部状态，从而推测出过去或未来的值。

## 攻击过程

1. **快速用户创建**：攻击者可以自动创建多个用户以获取三个令牌。
2. **暴力破解**：攻击者对JWT的HMAC签名进行暴力破解。  
   - 估计成本：8,000美元到24,000美元（使用云计算），尽管这需要较高的资源投入。
   - JWT的过期时间（7天）不会限制攻击，因为目标是用于签名的密码盐。

3. **重建密码盐**：成功破解后，攻击者可能重建出其他用户的密码盐，从而获取用户的JWT。
