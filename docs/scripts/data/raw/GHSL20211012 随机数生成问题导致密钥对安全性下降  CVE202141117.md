# GHSL-2021-1012: 随机数生成问题导致密钥对安全性下降 - CVE-2021-41117

## 技术基础知识

- **随机数生成**：在密码学中，生成随机数（尤其是密钥）是确保安全性的关键。
- **CSPRNG**（加密安全随机数生成器）：用于生成安全密码学密钥的随机数生成器。

## 问题概述

一个名为 `keypair` 的库在生成 RSA 密钥时，发现在某些情况下它产生的是相同的密钥。特别是，这种重复发出的密钥中的 `P` 和 `Q` 值是相同的。这在实际中是不应该发生的，意味着随机数生成过程存在严重问题。

### 具体问题

1. **库的随机数生成方法**：
   - 在浏览器环境中，它使用 `window.crypto.getRandomValues()` 来获取随机数据。
   - 在 Node.js 环境中，由于找不到 `window` 对象，库降级到使用一个较不安全的解决方案。

2. **随机数种子的生成**：
   - 使用了不当的方法给 AES-CMAC 生成器初始化种子，尤其是在 Node.js 环境中，这可能导致使用了 `Math.random()` 作为种子。

3. **编码错误**：
   - 随机数输出处理不当，转化为一个包括大量零的数组（例如，97% 的输出是 0），这样极大减少了密钥的随机性。

## 安全影响

由于生成的 RSA 密钥比较容易猜测，攻击者可能利用这一点解密机密信息或获得对受害者账户的授权访问。

## 发现与报告

该问题由 Axosoft 的工程师 Dan Suceava 发现，并报告给 GitHub 安全实验室。GitHub 安全工程师 Kevin Jones 独立调查并确认了问题的根源。
