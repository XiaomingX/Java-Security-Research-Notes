# GHSL-2020-295: ReDoS漏洞在is.js中的影响 - CVE-2020-26302

## 什么是ReDoS？

ReDoS（正则表达式拒绝服务）是一种安全漏洞，通常出现在设计不佳或效率低下的正则表达式中。当输入字符串构造得当时，正则表达式可能会表现得非常糟糕，导致极慢的运行时间。

## 漏洞影响

在is.js项目中，存在一个或多个易受ReDoS攻击的正则表达式。具体来说，该正则表达式在某些情况下的运行时间会达到O(2^n)，这意味着它会在特定输入下耗费大量计算资源。

## 造成ReDoS的原因

ReDoS通常由以下原因引起：
- **歧义**：正则表达式中的不同部分可能会导致匹配方式的重叠。
- **用户控制输入**：如果用户可以控制输入并导致正则表达式的慢速匹配，可能会导致服务拒绝。

## 修复方法

修复ReDoS问题的关键在于消除正则表达式内部的歧义和重叠。以下是一些示例：

### 示例1：HTML注释的匹配

**易受攻击的正则表达式**：
```regex
<!--.*?-->
```
这个正则表达式试图匹配HTML注释，因为`.`可以匹配任意字符，导致在处理包含多个空格的输入时效率低下。

**修复后的正则表达式**：
```regex
<!--[^-]*?-->
```
通过更改表达式，限制了匹配的范围，从而消除了歧义。

### 示例2：snake_case标识符

**易受攻击的正则表达式**：
```regex
(\w+_?)+
```
在这个例子中，正则表达式允许了多种方式匹配字母，也导致了匹配过程中的回溯问题。

**修复后的正则表达式**：
```regex
(\w+_)+\d+
```
这个修复避免了内部的重复匹配，令表达式更加具体，阻止了重叠。

## is.js中的具体情况

is.js复制了一个来自Gist的正则表达式来验证URL。如果利用恶意字符串进行验证，可能会导致正则表达式处于“永无止境”的循环中。

此漏洞是在使用CodeQL查询分析期间被发现的，使得该项目有可能引发拒绝服务攻击。
