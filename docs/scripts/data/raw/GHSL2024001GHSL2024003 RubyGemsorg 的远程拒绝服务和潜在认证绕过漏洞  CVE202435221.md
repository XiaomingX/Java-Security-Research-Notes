# GHSL-2024-001_GHSL-2024-003: RubyGems.org 的远程拒绝服务和潜在认证绕过漏洞 - CVE-2024-35221

在 RubyGems.org 发现了一个远程拒绝服务（DoS）漏洞和潜在的认证绕过漏洞。RubyGems.org 是 Ruby 社区的 gem 托管服务。

## 漏洞概述

### 1. 远程拒绝服务（DoS）漏洞

- **漏洞描述**：当 Gem 发布者发布 Gem 时，可能会导致远程拒绝服务。这是由于 Ruby 读取 Gem 文件的清单方式所致。使用 `Gem::Specification.from_yaml` 时，它会利用 `SafeYAML.load`，该方法允许在 Gem 的 YAML 元数据中使用 YAML 别名。这可能导致使用 YAML-bombs 的拒绝服务攻击（类似于亿万笑话攻击）。
  
- **影响**：攻击者可以通过向 RubyGems 服务器推送多个包含 YAML-bombs 的 gem 来占用所有工作进程，从而导致服务器完全失去响应。

### 2. 潜在的认证绕过漏洞

- **漏洞描述**：在某些罕见情况下，RubyGems 的信任发布控制器接受使用 "none" 算法的 JWT（JSON Web Token）作为有效的令牌。这可能允许攻击者用伪造的令牌替换有效的 API 密钥，从而发布 Gem。

- **修复建议**：通常情况下，JWT 库（json-jwt）的使用按预期工作，但如果提供给 `verify!` 方法的 JWKS（JSON Web Key Set）为 nil，将允许“none”算法的 JWT。

## 漏洞总结

- **影响范围**︰针对 RubyGems.org 的拒绝服务，攻击者需要通过发送特定请求来占用所有工作进程。
  
- **潜在风险**：在特定情况下，攻击者可能能够发布 Gem 或更改 Gem 的所有者，具体取决于 API 密钥的配置。
