# GHSL-2022-100: Autolab 中的路径遍历漏洞和远程代码执行漏洞

在 Autolab 中发现了两个漏洞，具体情况如下：

## 概述

### 受影响版本
- **Autolab**: v2.9.0

### 漏洞简介
1. **路径遍历漏洞 (CVE-2022-41955)**:
   - Autolab 允许学生提交作业，但其“远程提交”功能存在漏洞。未经身份验证的攻击者可以利用此功能提交作业，并访问注册用户的提交目录之外的文件。
   - 攻击者只需构造特定格式的 URL 并通过 GET 请求调用。这样可以在“提交历史”中查看或下载特定文件的内容（例如：/etc/passwd），从而导致信息泄露。

2. **远程代码执行漏洞 (CVE-2022-41956)**:
   - Autolab 为抄袭检查提供了 MOSS 功能，具备足够权限的老师可以运行此检查。某些参数在经过多个步骤时未经过滤，使得攻击者可能控制运行 Autolab 的服务器。
   - 攻击者只需使用浏览器访问特定链接，选中“最大行数”选项，并输入如 `| touch /tmp/pwned.txt |` 的命令，而不是有效的行数。这将创建一个文件 `/tmp/pwned.txt`。

## 漏洞影响
- **路径遍历漏洞**: 信息泄露
- **远程代码执行漏洞**: 攻击者可控制服务器
