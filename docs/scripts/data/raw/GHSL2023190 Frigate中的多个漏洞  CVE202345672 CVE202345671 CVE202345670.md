# GHSL-2023-190: Frigate中的多个漏洞 - CVE-2023-45672, CVE-2023-45671, CVE-2023-45670

Frigate是一个开源视频监控软件，存在多个安全漏洞，具体如下：

## 漏洞概述

1. **服务日志访问漏洞**:
   - **问题**: 通过`logs/<service>`接口，用户可以获取指定服务（如nginx、go2rtc、frigate）的日志。如果请求了一个不存在的服务，系统应该返回错误信息，但实际上却可能返回未清理的HTML格式数据。
   - **风险**: 可能导致JavaScript注入。

2. **代码执行漏洞**:
   - **现象**: 当用户提供的配置通过`load_config_with_no_duplicates`函数加载时，由于使用了可自定义构造函数（`yaml.loader.Loader`），攻击者可以执行任意代码。
   - **风险**: 可能导致未经身份验证的远程代码执行（RCE）。

3. **反射型XSS漏洞**:
   - **问题**: `recording_clip`请求处理器返回的字符串未经过清理，可能导致反射型跨站脚本攻击（XSS）。
   - **风险**: 攻击者可以构造恶意URL，以执行任意的JavaScript代码。

4. **缺乏CSRF保护**:
   - **现象**: `config_save`和`config_set`请求接口没有对跨站请求伪造（CSRF）的保护，允许来自任何来源的请求更新Frigate的配置。
   - **风险**: 攻击者可以通过构造恶意请求来修改服务器配置，导致服务中断和数据泄露。

## 具体案例

- 当请求不存在的摄像头时，系统返回的错误信息未经过滤，允许插入JavaScript代码。
- 使用以下请求会导致XSS攻击：
  ```
  GET /logs/<invalid_service>?payload=<malicious_code>
  ```
- 利用POST请求更新配置：
  ```html
  <form action="http://frg.server/config_save" method="POST">
      <input type="text" name="save_option" value="malicious_config"/>
  </form>
  ```

## 总结

Frigate存在多个安全漏洞，若不及时修复，可能导致攻击者执行任意代码、植入恶意脚本以及修改服务器配置。开发者应立即关注并采取措施来加固系统安全。
