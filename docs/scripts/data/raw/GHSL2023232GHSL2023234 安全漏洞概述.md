# GHSL-2023-232_GHSL-2023-234: 安全漏洞概述

在Flowise应用程序中发现了三个主要的安全漏洞：路径注入、跨站脚本（XSS）和CORS误配置。这些漏洞可能会危害服务器上信息的保密性，甚至允许攻击者读取服务器文件或修改用户秘密。

## 关键漏洞

1. **路径注入**
   - 在`/api/v1/openai-assistants-file`接口中，攻击者可以通过未经过滤的`fileName`参数读取任意文件。
   - 这可能导致**信息泄露**。

2. **跨站脚本（XSS）**
   - 多个API接口（如：`api/v1/chatflows/id`）中存在反射型XSS漏洞。
   - 如果使用默认的无身份验证配置，攻击者可以构造特殊的URL，将恶意JavaScript注入用户会话中，窃取信息或重定向用户。
   - 如果聊天流ID未找到，可能会在404页面中显示相关值，允许攻击者附加任意脚本。

3. **CORS误配置**
   - Access-Control-Allow-Origin设置为允许所有来源，导致任何网站可以请求Flowise，可能会窃取用户信息。
   - 此CORS配置也可能与路径注入结合，进一步放大安全风险。

## 发现方式
这些漏洞是通过**CodeQL**工具发现的，建议开发者对所有请求参数返回的数据进行严格的过滤处理。

## 风险总结
- **信息泄露**： 通过以上漏洞，攻击者可能获取敏感信息。
- **任意文件读取**： 攻击者可以读取Flowise服务器上的任意文件。
