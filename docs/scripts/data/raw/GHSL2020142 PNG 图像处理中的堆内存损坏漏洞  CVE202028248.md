# GHSL-2020-142: PNG 图像处理中的堆内存损坏漏洞 - CVE-2020-28248

## 背景
- **png-img** 是一个提供 libpng（用于处理 PNG 图像的库）绑定的包。
- 该包存在整数溢出漏洞，导致堆内存分配不足，最终可能导致堆内存损坏。

### 版本信息
- **当前版本**: png-img 2.3.0
- **包含的 libpng 版本**: 1.6.14（存在已知漏洞）
- **最新的 libpng 版本**: 1.6.37（强烈建议更新）

## 漏洞细节
1. **内存分配**:
   - png-img 在解析 PNG 图像时，使用 `PngImg::InitStorage` 函数分配内存。
   - 计算分配内存的大小时，使用了 `info_.height` 和 `info_.rowbytes` 进行乘法运算。

2. **整数溢出**:
   - `info_.height` 和 `info_.rowbytes` 是 32 位无符号整型。
   - 如果 `info_.height` 的值很大（例如：0x01000001）且 `info_.rowbytes` 为 0x100，则计算结果会造成溢出，导致实际分配的内存过小（0x100）。

3. **内存损坏**:
   - 由于分配的内存不足，生成的 `rowPtrs_` 数组会指向已分配内存的外部区域。
   - 这意味着程序在处理 PNG 文件时，可能会覆盖与 `data_` 区域相邻的内存，这可能被攻击者利用。

4. **执行任意代码的风险**:
   - 如果 png-img 用于处理不受信任的 PNG 文件，攻击者可以控制内存内容，这可能导致任意代码执行。

## 安全建议
- **更新 libpng**: 尽快将 libpng 更新至 1.6.37，以防止利用已知的漏洞。
