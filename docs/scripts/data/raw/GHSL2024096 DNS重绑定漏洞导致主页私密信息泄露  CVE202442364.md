# GHSL-2024-096: DNS重绑定漏洞导致主页私密信息泄露 - CVE-2024-42364

## 概述

主页（homepage）版本 0.9.1 的默认配置存在 DNS 重绑定漏洞，这使得攻击者可以访问主页拥有者的私密信息。

## 技术细节

- **DNS重绑定**：攻击者通过技术手段改变域名的DNS记录，从而让其网站能够访问其他内网资源。
  
- **攻击过程**：
  1. 攻击者让用户访问自己的网站。
  2. 当用户访问后，攻击者网站改变其域名的DNS记录，将其IP地址改为主页实例的内部IP地址。
  3. 使用子域名反绑定各个可能的IP地址，以检测哪些地址是有效的。
  4. 一旦找到有效的IP地址，攻击者就可以请求这个地址，获取主页实例的响应，而不是攻击者自己网站的响应。

- **信息泄露**：由于缺乏认证，攻击者可以读取用户的私密信息，如API密钥等。

## 风险

这个漏洞可能导致信息泄露，尤其是当主页实例和受害者在同一个内网时。
