# GHSL-2023-221: Digdag的路径遍历漏洞 - CVE-2024-25125

Treasure Data的Digdag工作负载自动化系统在配置为本地存储日志文件时，存在路径遍历漏洞。

## 漏洞概述

- **漏洞名称**: 路径遍历漏洞
- **产品**: Digdag
- **版本**: v0.10.5
- **影响**: 本地日志存储配置可以导致信息泄露

## 漏洞详情

1. **路径遍历漏洞**: 如果Digdag被配置为在本地存储日志文件（例如：通过命令行参数--task-log），就会受到这个漏洞的影响。
   
2. **攻击原理**:
   - 漏洞源自于API方法`getFile`的`filename`参数，该参数在经过访问权限和有效性检查后，传递到`getFile`方法。
   - 使用`LocalFileLogServerFactory`实例时，`getFile`方法会处理用户提供的文件名。
   - 恶意用户可以通过URL编码，传递诸如`%2fetc%2fpasswd`的绝对路径。由于`java.nio.file.Path`的`resolve`方法会忽略原始路径，这样就能绕过常规的过滤机制。

## 风险

- 该漏洞可能导致**信息泄露**，攻击者能够访问系统中的敏感文件。

## 发现与报告

- 此漏洞是由GHSL团队成员@p-（Peter Stöckli）发现并报告的。  
- 如果有任何有关此问题的沟通，请联系GHSL团队：securitylab@github.com，并提及“GHSL-2023-221”作为参考。

## 结论

为了保护系统安全，建议用户检查和更新配置，避免本地存储日志文件，或采取其他安全措施。
