# GHSL-2024-090: yt-dlp中的路径穿越漏洞导致的远程代码执行 - CVE-2024-38519

## 概述
`yt-dlp` 是一个下载视频和字幕的工具，但它不检查字幕的文件扩展名。这使得Windows用户在从特制链接下载带有字幕的视频时，容易受到路径穿越攻击，可能导致任意二进制文件被覆盖。

## 技术简介
- **yt-dlp**版本：`2024.04.09`
- **功能**：yt-dlp可以下载视频及其字幕，用户可通过参数指定下载字幕，如 `--write-subs`，`--write-auto-subs`，`--all-subs` 或 `--write-srt`。
  
### 处理字幕文件的方式
1. 每个支持的视频网站都有专用的提取器。
2. 某些提取器仅提供字幕的URL，yt-dlp会根据URL路径来解析扩展名。
3. 其他提取器不仅提供URL，还明确提供字幕扩展名。
    - 一些提取器会验证提取的扩展名，但有些不会。

### 如何构造文件名
- yt-dlp 根据提供的扩展名生成输出文件名：
  - 默认格式为 `%(title)s [%(id)s].%(sub_lang).%(sub_format)s`
  - `sub_lang` 默认为 `en`
  - `sub_format` 是提取出的字幕扩展名

### 漏洞影响
- 在Linux上，路径穿越攻击不会生效，因为Linux不接受类似 `not_existing_folder/../../file` 的路径。但在Windows上，这种攻击是有效的。
- 攻击者可以通过构造特定的字幕扩展名来利用此漏洞，最终可能导致未经授权的文件系统修改，甚至远程代码执行。

## 漏洞利用示例
攻击者可以在受支持的网站（如 youtube.com 或 bbc.co.uk）上利用该漏洞，但需要特定的条件：
- 网站可能在某一时刻被攻击者控制（例如通过XSS或SQL注入）。
- 攻击者要诱使用户下载带有恶意字幕的视频。
