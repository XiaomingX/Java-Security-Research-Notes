# Mealie中的SSRF和DoS漏洞概述

## 漏洞介绍
Mealie v1.0.0-RC1.1存在多个服务器端请求伪造（SSRF）和拒绝服务（DoS）漏洞。攻击者可以利用这些漏洞确定、映射和检索Mealie本地网络上的Web服务器内容，并可以对其他目标发起拒绝服务攻击。

## SSRF（服务器端请求伪造）
- **Safe_scrape_html函数**：该函数使用用户输入的URL向远程服务器发送请求。无人对URL进行限制，攻击者可以利用此功能识别本地网络上的HTTP(S)服务器。
- 处理响应时，如果目标URL返回有效响应但未返回食谱，则会显示400错误。如果没有响应或超时，则会显示500错误。
- 攻击者可以向`create-url` API端点发送直接请求，触发不同的响应。

## DoS（拒绝服务）
- **无速率限制**：函数请求没有速率限制，攻击者可以发送大量请求，导致Mealie容器的CPU资源耗尽。
- **大文件请求**：攻击者可以请求任意大文件（如Debian ISO），这将使Mealie的CPU占用过高，可能导致容器崩溃或重启。
- **反复请求**：攻击者可以持续发送请求，从而对Mealie或外部目标造成拒绝服务攻击。

## 图片请求漏洞
- **Scrape_image函数**：此函数根据用户提供的URL检索图像，没有进行有效性检查。
- 若请求的资源不存在，响应将为null；若请求有效但不是图像，也会返回相应的内容。
- 攻击者可以利用这个功能从具体服务器下载文件，即使是本地网络上的文件，从而可能导致管理员权限被获取。

## 安全风险
- 任何经过身份验证的用户都能利用这些漏洞评估Mealie服务访问的本地网络中的HTTP服务器。
- 缺乏速率限制可能导致Mealie容器崩溃，攻击者甚至可以控制对外部目标的网络流量。

## 结论
以上提到的漏洞（CVE-2024-31991至CVE-2024-31994）需要立即修复，以防止恶意用户利用这些漏洞导致系统损害。
