# Notepad++ Buffer Overflow Vulnerabilities

## 简介
Notepad++ 存在多个缓冲区溢出漏洞，允许恶意文件造成超出缓冲区边界的读写。以下是主要技术点的简化说明。

## 漏洞概览
- **受影响版本**: Notepad++ v8.5.2
- **相关CVE**: CVE-2023-40031, CVE-2023-40036, CVE-2023-40164, CVE-2023-40166

## 技术细节

### 1. UTF16到UTF8转换溢出
- **错误来源**: Utf8_16_Read::convert 函数分配的新缓冲区大小计算不正确，导致在处理某些奇数长度的输入时超出边界。
- **示例情况**:
  - 输入长度为9，计算出的新大小为14。
  - 处理8个字节后，指针移动，导致尝试访问第10个字节，这超出了输入长度。
- **后果**: 潜在的任意代码执行。

### 2. mCharToFreqOrder溢出
- **错误来源**: Notepad++ 使用的库在读取文件时，数组索引可能大于实际分配的大小，造成读取超出内存边界。
- **示例情况**:
  - 在POC中，设置的索引为5419，而实际大小为5376，导致错误读取。
- **后果**: 可能泄露内部内存分配的信息。

### 3. charLenTable溢出
- **错误来源**: charLenTable数组索引的设置可能发生错误，读取超出大小。
- **示例情况**:
  - 设置索引为9，但该数组仅有8个元素。
- **后果**: 可能泄露内部内存分配的信息。

### 4. 文件加载过程中的溢出
- **错误来源**: FileManager::loadFile 在文件加载时，读取超出缓冲区大小。
- **处理步骤**:
  - 加载数据时未检查边界，可能读取32个字节超出数据缓冲区。
- **后果**: 潜在的内存信息泄露。

## 漏洞重现
要测试这些漏洞，可以按照以下步骤：
1. 构建 Notepad++ 的 ASAN 版本。
2. 使用特制文件打开 Notepad++，观察是否出现超出边界的访问。
3. 使用 `GFlags` 启用页面堆，创建POC文件进行测试。
