# GHSL-2020-305: Regular Expression Denial of Service (ReDoS) - CVE-2020-26310

## 概述
这个项目包含一种或多种易受 ReDoS 攻击的正则表达式。ReDoS（正则表达式拒绝服务）是一种安全漏洞，通常出现在构造不当或低效的正则表达式中，这会导致它们在处理特定输入时性能极差。

## ReDoS 的原理
如果正则表达式具有模糊性或重叠，则可能导致指数级回溯，表现为 O(2^n) 的运行时间。当用户可以控制输入时，易受 ReDoS 攻击的正则表达式可能成为安全隐患。例如，在输入验证库中用于服务器验证不受信任的用户输入。

### 修复 ReDoS
解决 ReDoS 通常需要消除正则表达式中的模糊性和重叠，以下是示例及解决方案：

1. **HTML 注释示例：**
   - **易受攻击的正则表达式：**
     ```
     <!--.*?-->
     ```
   - **问题：** `.` 和 `\s` 都可以匹配空格，导致评估器在长空格时尝试多种匹配方式。
   - **修复方案：**
     ```
     <!--[^]*?-->
     ```
   - 这样可以明确匹配不含空白字符的内容，消除模糊性。

2. **snake_case 标识符示例：**
   - **易受攻击的正则表达式：**
     ```
     \w+(_\w+)*
     ```
   - **问题：** 字母的匹配方式导致多个分组可能性。
   - **修复方案：**
     ```
     \w+(_\w+)+
     ```
   - 指定每个组必须以下划线结束，减少模糊性。

3. **VS Code 中的例子：**
   - **易受攻击的正则表达式：**
     ```
     \S+
     ```
   - **修复方案：**
     ```
     [^\s\(\)]+
     ```
   - 通过更具体的匹配，去除与其他部分的重叠。

## 发现与报告
此漏洞通过 CodeQL 查询发现，识别出 HTML 解析器中使用的正则表达式存在漏洞。可导致拒绝服务的风险。
