# GHSL-2023-277: Owncast中的任意文件删除漏洞 (CVE-2024-31450)

## 概述
在版本0.1.2的Owncast应用程序中，拥有管理员权限的远程攻击者可以通过向 `/api/admin/emoji/delete` 发送恶意的POST请求，删除任意文件。

## 技术细节
- **Owncast版本**: 0.1.2
- **API接口**: Owncast提供了一个管理员API，位于 `/api/admin`。
- **漏洞位置**: `emoji/delete` 端点允许管理员删除自定义表情符号，这些符号保存在磁盘上。请求中的参数直接影响待删除文件的路径。

### 漏洞原理
1. **路径遍历攻击**: 通过在请求中使用路径遍历序列（如 `../`），攻击者可以绕过文件限制，从而删除系统上任意文件，而不仅仅是表情符号目录中的文件。
2. **需要管理员权限**: 此漏洞只对拥有管理员权限的用户有效。

### 攻击示例
为了利用这个漏洞，攻击者需要发送一个POST请求（包含所需的管理员凭证）：

```http
POST /api/admin/emoji/delete HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "../../path/to/file.txt"
}
```

## 发现与报告
- 该漏洞由GitHub CodeQL团队成员@atorralba (Tony Torralba) 发现，并使用CodeQL的路径注入查询进行了确认。
