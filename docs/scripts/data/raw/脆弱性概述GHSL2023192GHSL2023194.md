# 脆弱性概述：GHSL-2023-192_GHSL-2023-194

Bazarr 存在多个安全漏洞，编号为 CVE-2023-50264、CVE-2023-50265 和 CVE-2023-50266。这些漏洞允许未经身份验证的任意文件读取和盲目的服务器端请求伪造（SSRF）。

## 1. 关于 Bazarr
Bazarr 是一个管理和下载字幕的应用程序，版本为 v1.2.4，运行在 URL 地址 `http://127.0.0.1:6767` 上。

## 2. 漏洞详细信息

### 2.1 漏洞类型
- **任意文件读取**：访问特定的功能时，应用未能验证用户输入，导致攻击者能够读取服务器上任意文件。
- **盲 SSRF**：攻击者可以向服务器内部或外部资源发送请求，但无法直接获得响应，只有通过特定的 JSON 响应才能获取信息。

### 2.2 具体漏洞表现
- **漏洞一：** 
  - **位置**: `/system/backup/download/`
  - **描述**: 在 `bazarr/app/ui.py` 文件中，`backup_download` 方法未验证用户控制的文件名，导致任意文件读取。
  - **发现方式**: 使用 CodeQL 查询发现路径表达式中存在不受控制的数据。

- **漏洞二：** 
  - **位置**: `/api/swaggerui/static`
  - **描述**: 类似于上述漏洞，`swaggerui_static` 方法同样未对用户控制的文件名进行验证，导致任意文件读取。

- **漏洞三：** 
  - **位置**: 代理方法
  - **描述**: 在 `bazarr/bazarr/app/ui.py` 中，代理方法未验证用户提供的协议和 URL，这导致服务器端请求伪造。
  - **发现方式**: 使用 CodeQL 查询发现完整的服务器端请求伪造。

## 3. 漏洞影响
这些漏洞可能被攻击者利用，以便:
- 读取服务器上的敏感文件。
- 判断内部网络资源是否存在，尽管这些资源并不一定在互联网上可访问。
