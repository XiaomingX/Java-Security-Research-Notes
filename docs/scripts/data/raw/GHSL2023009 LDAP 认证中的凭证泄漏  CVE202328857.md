# GHSL-2023-009: LDAP 认证中的凭证泄漏 - CVE-2023-28857

## 概览

在用 X509 证书进行 LDAP 认证的 Apereo CAS 配置中，未经过身份验证的用户可以泄露 LDAP 认证的凭证。攻击者可以发送一个特别构造的 X509 客户端证书，该证书包含指向恶意资源的“CRL 分发点”扩展 URL。

## Apereo CAS

- 版本: v6.6.4

### X509 证书认证

Apereo CAS 可以配置使用基于客户 X509 证书的认证。这些证书可以通过 TLS 握手或特殊的 HTTP 头（如“ssl_client_cert”）提供。在检查客户端证书的有效性时，`X509CredentialsAuthenticationHandler` 会检查该证书是否被撤销。

- 撤销检查依赖于证书中的“CRL Distribution Points”扩展提供的 URL。这些 URL 由恶意用户控制。

### LDAP 认证和密码泄漏

如果 CAS 服务器配置为使用 LDAP 服务器进行 X509 认证，并设置了相关属性（如“cas.authn.x509.ldap.ldap-url”和“cas.authn.x509.ldap.bind-credential”），则 `X509CredentialsAuthenticationHandler` 会从证书中获取撤销 URL，可能是 LDAP URL。当 CAS 向这些 LDAP URL 发送请求时，它使用与初始 LDAP 服务器相同的密码，可能导致密码泄漏。

### 注意事项

- `X509CredentialsAuthenticationHandler` 仅检查证书的有效期和撤销状态，而不检查证书的签名有效性。因此，即使是不受信任的自签名证书也可以被利用。
- 如果 Apereo CAS 背后有反向代理，该代理验证 TLS 证书的有效性，此漏洞仍然可以被利用，但攻击者需要构造带有 CRLDP 扩展的签名证书。

## 示例配置

1. 下载 CAS Overlay v6.6.4（包括 `cas-server-support-x509` 和 `cas-server-support-rest-x509` 依赖）从 [cas-initializer](https://casinit.herokuapp.com/ui)。
2. 解压到本地文件夹。
3. 使用以下命令构建：
   ```bash
   ./gradlew clean build
   mkdir /etc/cas
   ./gradlew createKeystore
   ./gradlew copyCasConfiguration
   ```
4. 启动配置为具有 LDAP 连接的 CAS 实例，例如：`ldap://example.com:1389/`，并使用 `admin` 和 `s3cr3taaaaa` 凭证。此配置还启用 LDAP 撤销检查。

### 安全风险

未经身份验证的用户可以泄露服务器上配置的 LDAP 连接密码。
