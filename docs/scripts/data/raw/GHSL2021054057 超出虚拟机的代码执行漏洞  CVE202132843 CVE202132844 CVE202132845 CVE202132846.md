# GHSL-2021-054_057: 超出虚拟机的代码执行漏洞 - CVE-2021-32843, CVE-2021-32844, CVE-2021-32845, CVE-2021-32846

## 概述

恶意的客户机可以通过滥用某些驱动程序触发宿主机的漏洞，导致在虚拟机外部执行代码。

## 技术背景

### Hyperkit
Hyperkit是一个轻量级的虚拟化工具，使用`virtio`实现虚拟设备。`virtio`是一种标准的用于在虚拟机之间有效传递数据的接口。

### 结构体和函数指针
- `virtio_consts`: 包含虚拟设备使用的处理程序。
- `vc_cfgread`: 用于读取设备的配置，但如果没有检查调用该函数的指针是否为空，则可能导致宿主机崩溃。

### 漏洞情况
1. **vtrnd 设备**: 用于为客户机提供随机数。如果`vc_cfgread`没有检查为空，则可能导致宿主机崩溃，造成服务拒绝（DoS）。
2. **qnotify实现**: `pci_vtrnd_notify`没有检查`vq_getchain`的返回值，导致未初始化的`iovec`结构在读取内存时使用，从而可能导致宿主机崩溃或内存损坏。
3. **其他函数**: `pci_vtsock_proc_tx`有相似的漏洞，返回值检查不足可能导致更进一步的内存损坏。

## 漏洞利用示例
可通过启动一个多重引导内核来模拟受损的客户机，启动命令示例：
```
hyperkit -s 5,virtio-rnd
```
在尝试恢复这些漏洞时，建议使用调试器，而不是期待崩溃，因为崩溃未必会发生。

## 编译示例代码
要编译提供的示例代码：
1. 创建C文件，和`linker.ld`、`start.s`放在同一目录。
2. 安装`nasm`和`i686-elf-gcc`作为编译工具。

### 需要的文件
- `linker.ld`: 链接脚本
- `start.s`: 启动文件
