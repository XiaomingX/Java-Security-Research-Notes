# GHSL-2021-020: hbs 文件泄露漏洞 - CVE-2021-32822

## 概述

Express 的渲染 API 本应只传递模板数据。允许模板引擎的配置选项通过该 API 传递可能导致应用程序出现安全问题，包括跨站脚本（XSS）和远程代码执行（RCE）等。

## 技术基础知识

- **Express JS**：一个流行的 Node.js 框架，支持多种模板引擎，用于动态生成 HTML 响应。
- **模板引擎**：将模板中的占位符替换为应用程序提供的数据对象。
  
例如，以下代码渲染名为“index”的模板，并传递一个包含标题和消息的对象：

```javascript
res.render('index', { title: 'Hello', message: 'World' });
```

### 配置参数

模板引擎通常需要配置参数，如：
- 模板目录路径
- 模板名称
- 引擎特定参数

许多模板引擎选择直接通过 Express 渲染 API 接收这些配置选项，这就可能引发安全风险。

## 安全隐患

如果通过用户控制的对象（如 `req.query`）将配置参数直接传递到渲染引擎，攻击者就可能通过恶意的配置选项破坏应用程序的安全性。这种问题的严重性依据使用的引擎而异，从 XSS 到 RCE。

### 关键点

- **错误用法**：Express 的设计初衷是将模板数据和配置选项分开传递，但一些模板引擎混合这两者。
- **受影响的应用**：许多实际应用在使用不当的情况下面临漏洞风险。

## 示例代码

示例漏洞代码可以通过以下方式读取 vulnerable application 的 app.js 内容：

```javascript
// 漏洞实例
res.render('template', { config: { /* 恶意配置 */ }});
```

## 结论

这种 API 的误用在更广泛的情况下影响了许多应用程序。我们建议开发者遵循正确的 API 使用方式，以避免潜在的安全风险。
