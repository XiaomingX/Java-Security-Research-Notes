# GHSL-2024-029: Zammad中的拒绝服务（DoS）漏洞 - CVE-2024-33667

## 概述

在帮助台软件**Zammad**中发现了一个拒绝服务（DoS）漏洞。经过身份验证的攻击者可以阻止该网络应用程序处理任何请求。

## 受影响的版本

- Zammad v6.2.0

## 漏洞详情

### 如何发生的

1. **攻击方式**：经过身份验证的用户可以对Zammad进行远程拒绝服务攻击。这是通过调用某些特定的API端点实现的，例如导出日历（iCal格式）。
  
2. **代码流程**：
   - 攻击者通过URL中的方法参数调用到`CalendarSubscriptions`类中的一个名为`generic`的方法。
   - `generic`方法会调用另一个名为`generic_call`的方法，并将方法名称传递给它。
   - `generic_call`使用这个方法名称创建一个方法对象，并执行该方法。

3. **利用**：攻击者可以调用Ruby的**Kernel**模块中的`sleep`方法。如果调用此方法而不提供持续时间参数，它将导致当前请求线程永远处于休眠状态。

### 影响

- 如果Rails应用程序同时有多个线程和工作进程（例如：5个线程和2个工作进程），则发送10个请求将使应用程序无法响应其他请求。这可能会导致服务的拒绝，影响正常用户的使用。

### 漏洞利用示例

- 攻击者可以发送请求，将参数`params[:method]`设置为`sleep`。需要注意的是，需要用有效的（经过身份验证的）`_zammad_session_` cookie替换示例中的cookie值。
- 攻击者还可以引导Zammad帮助台工作人员访问其控制的网站，利用img标签等方式自动发送GET请求。

### 后果

- 该漏洞可能导致服务拒绝（DoS），并可能导致信息泄露（如果调用了某些返回敏感信息的方法）。
