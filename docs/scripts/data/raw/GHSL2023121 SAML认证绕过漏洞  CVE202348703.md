# GHSL-2023-121: SAML认证绕过漏洞 - CVE-2023-48703

## 漏洞简介
在RobotsAndPencils的go-saml库中发现了一个SAML认证绕过漏洞。这个问题可能导致使用go-saml进行SAML断言签名验证的应用程序出现认证绕过。

### 注意事项
- RobotsAndPencils/go-saml库已被归档，且不再维护。仍在使用该库的项目应迁移到其他SAML库，或移除SAML支持。
- 对于临时解决方案，参考“Workaround”。

## 漏洞详细信息
1. **漏洞原因**: 
   - 在内部调用`xmlsec1`工具进行SAML断言的签名验证时，没有指定验证所需的公钥数据。这允许攻击者自签名SAML断言并直接在SAML令牌中嵌入需要的公钥（如RSA密钥）。

2. **代码流程**:
   - 使用go-saml库验证SAML令牌时，通常会调用`Validate`方法。
   - `Validate`方法会再调用`VerifyResponseSignature`，而`VerifyResponseSignature`使用`xmlsec1`工具进行签名验证。

3. **易受攻击的代码路径**:
   - 如果代码路径在`VerifyResponseSignature`或`Validate`（在`authnresponse.go`中），或者在`VerifyRequestSignature`和`Validate`（在`authnrequest.go`中）中调用了私有验证函数，则该项目可能易受攻击。

### 临时解决方案
可以通过以下方式临时修复此漏洞：
- 复制go-saml项目，并在验证函数中调用`xmlsec1`时添加参数`--enabled-key-data`，并指定如`x509`或`raw-x509-cert`等值。
- 请参考pysaml2或xmlsec1的示例C代码。

### 版本更新
- `xmlsec1`在2024年1月发布的1.3.3版本中默认禁用了KeyValue和DEREncodedKeyValue XML节点。然而，大多数Linux发行版使用的是较旧版本，因此建议始终指定`enabled-key-data`。

## 结论
此漏洞可能导致使用go-saml进行认证的应用程序出现认证绕过。强烈建议迁移到其他维护中的SAML库。
