# GHSL-2021-058: CVE-2021-32847 漏洞概述

## 背景

- **技术名称**: hyperkit
- **版本**: v0.20210107

## 漏洞描述

一个恶意的虚拟机（guest）可以利用磁盘驱动中的一个漏洞来访问宿主机（host）的内存。这可能导致宿主机内存的泄露。

### 漏洞流程

1. **功能触发**: 函数 `pci_vtblk_proc` 处理虚拟机发送的磁盘 I/O 操作（使用 virtio）。 
2. **漏洞类型**: `VBH_OP_DISCARD` 操作存在漏洞，允许攻击者读取虚拟机地址空间之外的内存。
3. **问题所在**: 在代码中的断言（assert）检查 `iov` 数据结构的大小时，出现了一个拼写错误。错误地将相等操作符替换为赋值操作符，导致断言总是被满足。

### 漏洞利用

- 恶意虚拟机可以通过将磁盘镜像设置为已知状态并触发漏洞，利用无效的 `iov[1].iov_base` 指针读取外部内存。
- 攻击者可以精确控制被读取的扇区数量，逐步获取泄露的数据。

### 攻击流程

- 攻击者准备一个特殊的 `virtio_blk_discard_write_zeroes` 结构，使得泄露的数据按较小的增量填充，避免写入大量未知的扇区耗时过长。
- 由于测试中，guest 的内存初始化在 MALLOC_LARGE 区域，位于 MALLOC_TINY 之前，这使得 heap 操作尤为容易。

### 漏洞影响

该漏洞可能导致恶意虚拟机读取宿主机的重要内存数据。
