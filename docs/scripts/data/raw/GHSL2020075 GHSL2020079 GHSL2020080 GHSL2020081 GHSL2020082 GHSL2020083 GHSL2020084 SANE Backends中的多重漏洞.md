# GHSL-2020-075, GHSL-2020-079, GHSL-2020-080, GHSL-2020-081, GHSL-2020-082, GHSL-2020-083, GHSL-2020-084: SANE Backends中的多重漏洞

## 概述
SANE Backends存在多个内存破坏漏洞，这些漏洞可以被连接到同一网络的恶意设备或计算机触发。当应用程序（如simple-scan）搜索网络上的扫描仪时，就可能触发这些漏洞。例如，simple-scan在启动时立即尝试查找扫描仪，这样用户甚至不需要误以为连接了恶意扫描仪。

## 漏洞类型
1. **远程拒绝服务（DoS）:** 
   - 当简单扫描启动时，如果连接了恶意设备，应用程序可能会崩溃。
   - 漏洞发生在`sanei_epson_net_read`和`epsonds_net_read`函数中，均因处理意外大小的响应时存在错误。

2. **远程代码执行（RCE）:** 
   - 攻击者可以通过控制数据大小来触发堆缓冲区溢出，这可能导致代码被执行，尤其是在`epsonds_net_read`中。

3. **信息泄露:** 
   - 函数`esci2_check_header`存在信息泄露漏洞，允许攻击者通过缓冲区溢出读取并利用数据。
   - 此问题在用户点击“扫描”按钮时触发，因此属于“一次点击”漏洞。

## 具体漏洞
- **sanei_epson_net_read**：在处理响应大小时可能导致空指针解引用，导致程序崩溃。
- **epsonds_net_read**：
  - 存在堆缓冲区溢出，攻击者可控制写入堆缓冲区的数据。
  - 代码在处理未初始化数据时可能导致风险。
- **decode_binary** 和 **esci2_check_header**：可以导致读取越界，增加潜在漏洞的可利用性。

## 漏洞影响
这些漏洞意味着恶意用户只需连接到同一网络即可对SANE Backends造成影响，运行在受影响版本（如`libsane1 1.0.27-1`）的Ubuntu系统将面临风险。
