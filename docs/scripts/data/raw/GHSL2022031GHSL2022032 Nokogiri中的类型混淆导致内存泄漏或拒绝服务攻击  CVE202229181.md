# GHSL-2022-031_GHSL-2022-032: Nokogiri中的类型混淆导致内存泄漏或拒绝服务攻击 - CVE-2022-29181

## 概述

在处理恶意数据时，Nokogiri 存在两个类型混淆问题，这可能导致内存泄漏或拒绝服务攻击（DoS）。

## 问题描述

### 影响版本

- **Nokogiri**: v1.13.4

### 漏洞细节

攻击者可以控制传入 `parse_memory` 函数的对象类型，提供非字符串类型的对象（例如整数）。整数作为立即数，而不是对象的指针，导致程序在试图读取无效内存地址时崩溃。

### 示例代码

在 `xml_sax_parser_context.c` 文件的第68行，`RSTRING_LEN` 函数被使用，但没有检查数据的类型是否为字符串（T_STRING）。通常应该使用 `Check_Type(data, T_STRING)` 来进行检查，但在这里未进行检查。

## 潜在影响

1. **拒绝服务（DoS）**: 攻击者可以通过传递恶意对象来让程序崩溃。
2. **信息泄露**: 通过反复崩溃程序，攻击者可以猜测出有效的内存地址，进而访问本应保护的字符串内容。
