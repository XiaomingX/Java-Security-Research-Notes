# GHSL-2023-232_GHSL-2023-234: Flowise 安全漏洞简述

Flowise 存在以下几种安全漏洞，可能会影响服务器上的信息安全。这些漏洞包括路径注入、跨站脚本（XSS）和跨源资源共享（CORS）配置错误，最严重的情况下，攻击者可以从 Flowise 服务器读取文件并访问/修改用户密钥。

## 漏洞类型

### 1. 路径注入
- **问题**: 在 `index.ts` 中，`/api/v1/openai-assistants-file` 端点缺乏对 `fileName` 参数的有效性检查，导致任意文件读取漏洞。
- **影响**: 攻击者可以读取服务器上的任意文件。

### 2. 跨站脚本 (XSS)
- **问题**: 多个 API 端点存在反射 XSS 漏洞，如使用默认未经身份验证的配置时，攻击者可以构造特制的 URL，将 JavaScript 代码注入用户会话。
- **影响**: 攻击者可以窃取用户信息、创建虚假弹窗或重定向用户到其他网站。
- **特别情况**: 如果未找到 `chatflow ID`，其值会反馈在 404 页面，并允许攻击者附加任意脚本。

### 3. CORS 配置错误
- **问题**: CORS 设置将 `Access-Control-Allow-Origin` 头部设置为所有来源，允许任何来源访问 Flowise。
- **影响**: 在默认配置下，未经身份验证的任意来源可以请求 Flowise，从而窃取用户信息。

## 推荐措施
- **清理输入**: 建议在所有处理请求的端点中，确保从 `req` 参数传递到 `res` 的数据经过有效性检查和清理。示例代码应该移除所有未经处理的数据传递。
