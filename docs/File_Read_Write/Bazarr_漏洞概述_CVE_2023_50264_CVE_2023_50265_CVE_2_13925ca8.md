# Bazarr 漏洞概述: CVE-2023-50264, CVE-2023-50265, CVE-2023-50266

Bazarr 存在多个漏洞，主要包括未授权的任意文件读取和盲目服务器端请求伪造（SSRF）。

## 漏洞详细信息

1. **未授权的任意文件读取**
    - **漏洞位置**: `/system/backup/download/` 端点
    - **问题描述**: Bazarr 没有验证用户输入的文件名，通过 `send_file` 函数读取任意文件。
    - **影响版本**: v1.2.4
    - **检测方法**: 使用 CodeQL 查询识别的“未控制的数据用于路径表达式”。
   
2. **另一个文件读取漏洞**
    - **漏洞位置**: `/api/swaggerui/static` 端点
    - **问题描述**: 同样，用户输入的文件名未经过验证，允许任意文件读取。
    - **检测方法**: 使用 CodeQL 查询识别的“未控制的数据用于路径表达式”。

3. **盲目服务器端请求伪造（SSRF）**
    - **漏洞位置**: `proxy` 方法
    - **问题描述**: Bazarr 未验证用户输入的协议和 URL，直接将其传递给 `requests.get()`，造成服务器端请求伪造。请求的响应内容对于请求者是不可见的。
    - **风险**: 可能用于探测内部网络中的资源，即使它们在互联网上不可访问。
