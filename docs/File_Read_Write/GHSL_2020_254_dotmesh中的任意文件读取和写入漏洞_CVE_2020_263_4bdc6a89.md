# GHSL-2020-254: dotmesh中的任意文件读取和写入漏洞 - CVE-2020-26312

## 概述
在dotmesh中，由于不安全的符号链接处理，攻击者可以读取和/或写入目标文件夹外的任意位置。

## 技术细节

- **漏洞原因**: 在解压tar档案时，`untarFile` 函数本意是防止创建指向档案外部的符号链接。但攻击者可以利用一个特殊制作的tar档案，创建指向档案父目录的符号链接。
  
- **利用方式**:
  1. 攻击者的tar档案首先创建一个指向上层目录的符号链接（例如：`subdir/parent`指向`..`）。
  2. 然后创建另一个符号链接（例如：`subdir/parent/escapes`指向`..`），导致这个链接指向tar档案的父目录。

- **后果**:
  - **任意文件写入**: 如果攻击者可以控制档案文件，他们可以写入任意文件，权限与运行解压操作的程序相同。
  - **任意文件读取**: 如果攻击者可以读取解压后的文件，他们可能会读取系统中任意文件，只要该文件的权限允许读取。
