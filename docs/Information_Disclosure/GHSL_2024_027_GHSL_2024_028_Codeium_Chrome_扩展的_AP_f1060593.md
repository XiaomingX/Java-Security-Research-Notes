# GHSL-2024-027_GHSL-2024-028: Codeium-Chrome 扩展的 API 滥用 - CVE-2024-28120

## 概述
Codeium-Chrome 扩展存在一个安全漏洞，具体表现为其服务工作者在接收外部消息时没有验证消息的发送者。这意味着攻击者可以利用此漏洞获取用户的 Codeium API 密钥，从而冒充用户在后台自动完成功能上进行操作。

## 关键技术点
- **服务工作者**：Codeium-Chrome 扩展中的一部分，它负责处理后台任务和消息。
- **外部消息**：指的是来自其他网站或扩展的消息。
- **API 密钥**：用于身份验证的密钥，允许用户与 Codeium 自动填写服务器进行交互。

## 漏洞细节
- **版本**：受影响的扩展版本为 v1.2.52。
- **信息泄露**：攻击者可以通过未验证的外部消息获取用户的用户名、API 密钥及其他配置信息。
- **利用方式**：攻击者只需创建一个含有特定 JavaScript 代码的网页，即可向 Codeium-Chrome 扩展发送消息并显示 API 密钥和用户名。

## 安全建议
- 开发者需要在服务工作者中添加发送者验证逻辑，以防止未授权访问。
