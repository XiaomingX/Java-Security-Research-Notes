# GHSL-2021-058: 虚拟化环境中主机内存泄露漏洞 - CVE-2021-32847

## 概述
在虚拟化环境中，恶意的虚拟机（guest）可能会利用一个漏洞，通过磁盘驱动程序泄露主机（host）的内存。该漏洞影响了 **hyperkit** 版本 v0.20210107。

## 技术细节
### 漏洞原因
1. **功能描述**: `pci_vtblk_proc` 函数处理来自虚拟机的磁盘I/O操作请求。操作 `VBH_OP_DISCARD` 存在安全漏洞，允许攻击者读取虚拟机以外的内存。
2. **代码错误**: 在 `pci_virtio_block.c` 文件中，错误地将等式操作符 `==` 写成了赋值操作符 `=`。这导致一个断言总是返回真，忽略了输入数据的大小检查。

### 漏洞利用
- 恶意虚拟机通过先设置已知状态的磁盘镜像，再触发该漏洞，能够利用无效的内存指针 `iov[1].iov_base` 来访问主机内存。
- 攻击者可以控制 `num_sectors` 字段，从而读取被覆盖的内存字节。

### 实践影响
这个漏洞的影响更严重，因为在测试中，虚拟机的内存位置恰好与主机的其他内存区域相连，可能导致恶意代码操控内存。

## 复现漏洞
- **准备工作**: 
  - 编译 hyperkit 时，需启用 `AddressSanitizer` 以便检测无效内存读取。
  - 利用示例代码创建多启动内核，虚拟机需启用 `virtio-blk` 设备。

### 编译步骤
1. 将示例代码放入一个 C 文件，并准备以下文件：
   - `linker.ld`
   - `start.s`
2. 安装必要的编译工具：
   - `nasm`
   - `i686-elf-gcc`
