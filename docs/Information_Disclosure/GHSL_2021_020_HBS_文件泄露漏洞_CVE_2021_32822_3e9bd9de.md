# GHSL-2021-020: HBS 文件泄露漏洞 - CVE-2021-32822

## 概述

Express JS 的渲染 API 本来只应传递模板数据。但是，如果允许传递模板引擎的配置选项，可能会导致安全问题，包括跨站脚本（XSS）和远程代码执行（RCE）。

## 背景知识

- **Express JS**：一个流行的 web 框架，允许使用不同的模板引擎。
- **模板引擎**：用于处理动态内容的工具，它们通过检查传入的对象来替换模板中的内容。

### 示例

例如，可以使用以下代码渲染名为“index”的模板，并传递一个包含标题和消息的对象：

```javascript
res.render('index', { title: 'Hello', message: 'World' });
```

## 问题描述

许多模板引擎希望直接通过 Express 的渲染 API 接收其配置参数（例如模板目录路径和模板名称）。这可能会导致安全漏洞，尤其是当传递的对象是用户可控时（例如，`req.query`）。这样，恶意用户可能会通过提供不安全的配置选项来操纵应用程序。

### 安全影响

- 影响范围：取决于应用程序使用的模板引擎，可能包括 XSS 和 RCE。
- **重要**：这种混合使用（模板数据和配置选项）并不是 Express 的设计初衷。

## 具体案例

以 hbs（v4.1.1）为例，它不当混合了模板数据和配置选项，可能导致文件泄露漏洞。通过覆盖内部配置选项，不当的应用程序代码可能导致泄露敏感信息。

## 结论

我们的研究发现，很多模板引擎不经意间使用了这种不安全的 Express 渲染 API 模式，导致许多下游应用程序可能受影响。我们向开发者发出警告，以防止未来类似的 API 误用。
