# GHSL-2022-031_GHSL-2022-032: Nokogiri中的类型混淆导致内存泄漏或拒绝服务 - CVE-2022-29181

## 概述
Nokogiri库在处理恶意数据时存在两个类型混淆的问题。这些问题可能导致内存内容泄露或拒绝服务（DoS）攻击。

## 影响版本
- **Nokogiri v1.13.4**

## 问题描述
攻击者可以控制传递给`parse_memory`函数的对象类型，允许传入非字符串对象，例如整数。由于整数作为即时值而不是指针，尝试访问无效的内存地址时，进程会崩溃。

### 技术细节
以下的代码片段展示了一个问题：
```c
// 从 xml_sax_parser_context.c#L68 中获取的片段
RSTRING_LEN(data);
```
在这里，代码没有检查传入对象的类型是否为 T_STRING。通常情况下，应该使用`Check_Type(data, T_STRING)`来进行检查，但在此代码中并未执行任何检查。

### 漏洞影响
- **拒绝服务（DoS）：** 攻击者可以通过反复崩溃服务器，猜测有效的内存地址。
- **信息泄露：** 如果Nokogiri的解析结果被返回给攻击者，可能会泄露到内存中的任意Ruby字符串，包括机密信息。

## 其他影响
在html4版本的解析器中也发现了同样的漏洞：
```c
// 从 html4_sax_parser_context.c#L25 中获取的片段
```
