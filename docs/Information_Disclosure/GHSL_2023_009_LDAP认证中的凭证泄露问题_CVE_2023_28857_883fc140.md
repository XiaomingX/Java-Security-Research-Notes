# GHSL-2023-009: LDAP认证中的凭证泄露问题 - CVE-2023-28857

## 概述

在Apereo CAS（一个单点登录解决方案）通过LDAP目录进行X509证书认证时，未认证的用户可以泄露LDAP的凭证。攻击者可以发送特制的X509客户端证书，利用证书中的“CRL Distribution Points”扩展获取指向恶意资源的URLs。

## 技术背景

- **Apereo CAS 版本**: v6.6.4
- **X509证书认证**: Apereo CAS可以使用客户端X509证书进行认证，这些证书可以通过TLS握手或特殊的HTTP头（例如“ssl_client_cert”）提供。
- **证书验证**: X509CredentialsAuthenticationHandler会检查提供的证书是否未被撤销（revoked）。这个检查是通过从证书中提取的“CRL Distribution Points”获取撤销URLs来实现。

## 漏洞成因

1. **恶意URLs**: 如果证书中包含恶意的CRL URL，CAS将访问这些URL。
2. **密码泄露**: 在CAS配置使用LDAP进行X509认证时，CAS会使用初始配置的LDAP服务器密码进行请求。如果请求的LDAP URL是恶意的，则此密码可能会泄露。
3. **签名有效性问题**: X509CredentialsAuthenticationHandler只检查证书的日期和撤销状态，而不检查证书的签名。因此，即使是未受信任的自签名证书也可被利用。

## 配置示例

可以从[cas-initializer](https://casinit.herokuapp.com/ui)下载CAS Overlay v6.6.4，包含必要的依赖项（cas-server-support-x509 和 cas-server-support-rest-x509），并按照下列命令构建：

```bash
./gradlew clean build
mkdir /etc/cas
./gradlew createKeystore
./gradlew copyCasConfiguration
```

示例配置将CAS实例设置为使用位于`ldap://example.com:1389/`的LDAP服务器，并使用`admin`和`s3cr3taaaaa`凭证，同时启用LDAP的撤销检查。

### 注意
此示例配置只是便于测试，生产环境中的实际配置可能不同，但依然存在漏洞。

## 风险

- 任何未认证用户都可以泄露与服务器LDAP连接的密码。
