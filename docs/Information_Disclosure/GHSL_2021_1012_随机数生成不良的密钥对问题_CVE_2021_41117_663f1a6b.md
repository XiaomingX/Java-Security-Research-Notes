# GHSL-2021-1012: 随机数生成不良的密钥对问题 - CVE-2021-41117

## 背景

`keypair` 库实现了多种加密原语，部分依赖其他库（如 node-forge）。研究发现，这个库在生成SSH用的RSA密钥时，产生了相同的密钥。这意味着产生的 P 和 Q 值（以及 N 值）是相同的，这在实际应用中是不可接受的，特别是对于RSA-2048密钥。

问题的根源在于随机数生成不良，表明其对CSPRNG（加密安全伪随机数生成器）输出处理不当。

## 技术问题

### 版本
- `keypair` 版本：v1.0.3

### 随机数生成方法

- 库没有完全依赖平台提供的CSPRNG，而是使用了基于计数器的CMAC方法。
- 在浏览器环境下，它使用 `window.crypto.getRandomValues()` 来生成“真正”的随机数据。
- 在Node.js环境下，库无法找到`window`对象，因此使用了不那么安全的替代方案，且这个方案本身也存在bug。

### 具体问题

1. **CSPRNG调用错误**: 
   - 原本应该调用Node的CSPRNG，但由于变量命名冲突，导致这个调用分支从未被执行。

2. **随机数生成算法**:
   - 当`window.crypto.getRandomValues()`不可用时，库使用Lehmer LCG（线性同余生成器）生成随机数，且种子来自于 `Math.random()`，这本身就是一个安全隐患。
   
3. **输出掩码错误**:
   - 生成的输出经过了错误的编码，具体问题在于下面的代码行：
     ```javascript
     String.fromCharCode(String.fromCharCode(next & 0xFF))
     ```
   - 上述代码的双重 `String.fromCharCode` 是无意的，导致大部分输出为零。结果是97%的种子输出是0，只有0到9之间的数字是有效值。

## 影响

1. **安全风险**: 
   - 由于随机数生成不良，`keypair` 生成的RSA密钥容易被猜测，攻击者可能会解密机密信息或获授权访问受害者的账户。

2. **责任和发现**:
   - 该问题由Axosoft的Ross Wheeler报告，工程师Dan Suceava首次发现影响，GitHub安全工程师Kevin Jones独立调查并确认了bug的根源。
