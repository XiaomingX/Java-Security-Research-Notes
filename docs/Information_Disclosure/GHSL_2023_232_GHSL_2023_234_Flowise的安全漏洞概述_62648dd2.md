# GHSL-2023-232_GHSL-2023-234: Flowise的安全漏洞概述

## 漏洞类型
1. **路径注入**（Path Injection）
2. **跨站脚本攻击**（Cross-Site Scripting，XSS）
3. **CORS配置错误**

## 涉及的漏洞
- CVE-2024-36420
- CVE-2024-36421
- CVE-2024-36422
- CVE-2024-36423
- CVE-2024-37145
- CVE-2024-37146

## 什么是Flowise？
Flowise是一个管理聊天流的工具，存在多个安全漏洞，可能会导致服务器上的信息泄露。

## 漏洞详情

### 1. 路径注入漏洞
- **影响版本**：v1.4.3
- **具体位置**：`/api/v1/openai-assistants-file`
- **问题**：`fileName`参数没有经过清理，可能允许攻击者读取任意文件。

### 2. 跨站脚本（XSS）
- **问题描述**：许多API接口存在反射型XSS漏洞。如果使用默认的未认证配置，攻击者可以构造特定的URL，注入JavaScript脚本，执行恶意操作（如盗取信息、弹出假消息、重定向用户等）。
- **影响范围**：多个接口，包括：
    - `/api/v1/chatflows/id`
    - `/api/v1/public-chatflows/id`
    - `/api/v1/chatflows-streaming/id`
    - `/api/v1/credentials/id`

### 3. CORS配置错误
- **问题**：CORS的`Access-Control-Allow-Origin`头设置为所有，这允许任意来源访问Flowise。
- **后果**：在默认配置下，攻击者可以从Flowise请求用户信息。

## 发现和建议
- **发现者**：GHSL团队成员 @Kwstubbs
- **建议**：移除所有未经清理的数据直接传递到响应的情况，确保安全。
