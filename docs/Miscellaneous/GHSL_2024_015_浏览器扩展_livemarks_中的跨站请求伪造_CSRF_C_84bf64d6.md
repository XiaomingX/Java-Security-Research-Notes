# GHSL-2024-015: 浏览器扩展 livemarks 中的跨站请求伪造 (CSRF) - CVE-2024-30252

## 概述

livemarks 浏览器扩展存在 CSRF 漏洞。恶意网站可以诱使此扩展向任意 URL 发送已认证的 GET 请求。

### 主要组件

- **livemarks 版本**: 3.6
- **关键脚本**: 
    - `subscribe.js`: 处理 RSS 订阅，使用当前 URL 的参数作为 RSS 源，并验证其是否为有效的 XML。
    - `feed-parser.js`: 用于解析 RSS 数据。
- **权限配置**: `manifest.json` 文件定义了哪些资源可以被外部网站访问。

## 漏洞影响

该漏洞可能导致特权升级。盲目 CSRF 攻击会破坏在私有网络上运行服务器的完整性。用户可能在浏览器扩展中有一个私有服务器，假定它是安全的，但如果接收到来自攻击者的认证请求，安全性将受到威胁。

## 利用此漏洞的条件

- **UUID 泄露**: 攻击者需要获取扩展的 UUID 信息，该扩展仅适用于 Firefox。
- **操作方式**: 
    - 攻击者可以诱骗用户安装附加扩展，这个扩展具备某些权限（如管理权限），但不直接允许发送认证请求。
    - 一旦 UUID 泄露，攻击者可以通过 CSRF 攻击发送 SameSite cookie（因为 livemarks 对所有 URL 拥有权限）。

## 攻击示例

1. **托管恶意 HTML**: 攻击者托管一个 HTML 页面。
2. **诱导用户访问**: 用户访问该 HTML 页面，并激活 livemarks 扩展，最终导致向攻击者提供的域名发送已认证的请求。
