# GHSL-2024-029: Zammad软件中的服务拒绝（DoS）漏洞 - CVE-2024-33667

## 概述

在帮助台软件Zammad中发现了一个服务拒绝（DoS）漏洞。经过身份验证的攻击者可以阻止该网络应用程序处理任何请求。

## 受影响的软件版本

- Zammad v6.2.0

## 漏洞详情

1. **攻击方式**：
   - 经过身份验证的用户可以执行远程服务拒绝攻击。
   - Zammad提供导出帮助台工作人员日历的接口，格式为iCal。

2. **技术原理**：
   - 使用的参数通过URL传递，并进入`CalendarSubscriptions`类中的`generic`方法。
   - `generic`方法调用`generic_call`，后者根据方法名创建一个并调用方法对象。
   - Ruby允许方法对象引用任何可用的方法，包括父类中的方法（如Object和BasicObject）。
   - `Kernel`模块的方法在所有Ruby对象中可用。

3. **漏洞利用**：
   - 大多数关于这种漏洞的公开信息提到了一些破坏性的方法（如`exit`和`exit!`）。
   - 但研究发现，使用`Kernel`中的`sleep`方法可能更有效。
   - 传递不带持续时间参数的`sleep`会使当前请求线程永远挂起。
   - 比如，如果Rails应用最多启动5个线程（RAILS_MAX_THREADS=5）和2个Puma工作进程（WEB_CONCURRENCY=2），那么发送10个请求将使应用不再响应。

4. **攻击示例**：
   - 通过设置`params[:method]`为`sleep`来触发这个漏洞。
   - 提示：用有效的（经过身份验证的）`_zammad_session_` cookie替换请求中的值。

5. **攻击途径**：
   - 攻击者可以引导Zammad工作人员访问他们控制的网站，该网站会调用该接口（例如，通过在`img`标签中加载）。
   - 由于GET请求足以触发该漏洞，因此经过身份验证的cookie会自动由浏览器发送。

## 风险

- 此漏洞可能导致服务拒绝（DoS）。
- 如果可以调用的方法返回信息，可能导致信息泄露。
