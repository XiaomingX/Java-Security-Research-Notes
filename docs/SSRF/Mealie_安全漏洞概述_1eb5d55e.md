# Mealie 安全漏洞概述

## 漏洞信息
Mealie 版本 1.0.0-RC1.1 存在多个服务器端请求伪造（SSRF）和拒绝服务（DoS）漏洞。这些漏洞使得攻击者能够：
- 识别并访问 Mealie 本地网络上的 web 服务器
- 对目标发起拒绝服务攻击

## 技术背景
### SSRF（服务器端请求伪造）
- Mealie 的 `safe_scrape_html` 函数允许用户提供 URL 来向远程服务器发起请求。这些请求没有限制，可使用任意 IP 地址。
- 当请求返回任何响应时，攻击者可确认本地网络中的 HTTP 服务器。

### DoS（拒绝服务）
- `safe_scrape_html` 和 `scrape_image` 两个函数没有实施请求速率限制，可能导致 CPU 资源耗尽。
- 如果向 Mealie 请求大型文件（例如 ISO 文件），可以使容器 CPU 达到满载，导致服务不可用。

## 安全风险
1. **请求内容的处理**:
   - 用户提供的 URL 不被验证，可能指向本地或恶意的资源。
   - 向 Mealie 发送不存在的资源请求时，可能会返回错误消息，但攻击者可以利用这些信息来确认资源是否存在。

2. **资源耗尽**:
   - 攻击者可以持续向 Mealie 发起大量请求，导致 CPU 和内存耗尽，最终导致服务崩溃。

3. **文件存储和访问**:
   - Mealie 在请求图像时，未限制资源请求路径，可能导致敏感文件被意外下载和存储。
   - 有管理员权限的用户可以访问和下载这些文件，增加安全隐患。

## 预防措施
- **限制用户请求**: 对用户输入的 URL 进行验证，确保只能访问授权的外部资源。
- **实施速率限制**: 对请求限制频率，以防止恶意用户发起 DoS 攻击。
- **加强用户权限管理**: 确保只有授权用户才能访问和操作敏感数据。
