# GHSL-2023-185: Posthog中的服务器端请求伪造（SSRF） - CVE-2023-46746

## 什么是SSRF？

服务器端请求伪造（SSRF）是一种安全漏洞，攻击者可以利用它从服务器发送伪造的请求。这通常会导致敏感信息泄露或对内部系统的攻击。

## 问题描述

在Posthog的某个版本（1.43.1）中，发现了一个可以被经过身份验证的用户利用的SSRF漏洞。

具体问题在于，`posthog/api/user.py`文件中的`test_slack_webhook`方法没有正确验证用户提供的webhook变量。这使得恶意用户可以利用这个变量，导致Posthog服务器发送错误的POST请求。

这个漏洞是通过CodeQL的请求伪造查询发现的。

## 风险

通过这个SSRF漏洞，攻击者可以伪造从PostHog服务器发送的POST请求，可能会影响系统的安全性。
