# Rust 编译器演变：LLVM 和 GCC 的故事

## 简介

Rust 是一种能够 **保证内存安全且不牺牲性能** 的编程语言，因此在软件开发人员中受到了广泛欢迎。从以前的“小众”工具变成现在应用于 **Windows、Android** 等系统的重要语言，Rust 的迅速普及与它的编译器技术密不可分。

Rust 的主要编译器之一是 `rustc`，它使用 **LLVM** 作为后端编译器技术。但最近也出现了一些新变化，Rust 社区正在开发另一个新的编译器前端，称为 **gccrs**，它是基于 **GCC** 的。本文将带你了解这些不同的编译器，以及它们背后的技术区别。

## 什么是 LLVM？

**LLVM** 是一组编译器工具的集合，用于生成高效的机器代码。简单来说，它帮助编程语言将代码编译成不同计算机可以直接理解的低级代码。它名字的全称是 **Low-Level Virtual Machine**，但现在更像是一个品牌名称。

LLVM 的最大特点是它 **非常模块化**。比如，你可以用它来支持很多不同的编程语言——这也正是 Rust 编译器 `rustc` 选择它作为后端的原因。

举个例子，假如你开发了一款手机应用，LLVM 能帮你把代码优化得非常高效，无论你编写的代码用的是什么语言，应用是要跑在 iPhone 还是 Android 上。

## 什么是 GCC？

**GCC** 是 **GNU 编译器集合**，它是一个老牌的开源编译器，支持 **C、C++、Fortran** 等多种编程语言。GCC 的特别之处在于它非常 **稳定可靠**，可以支持非常多的不同架构和操作系统。

为了给更多的平台提供编译支持，GCC 开始支持新的语言，比如 Rust。这个新前端被称为 **gccrs**，它是 GCC 项目中的一个部分，目前还在开发中。

GCC 和 LLVM 之间的主要区别在于：**LLVM 更加模块化**，设计时就考虑了可扩展性，可以方便地添加支持新的编程语言。而 GCC 更像是一个整体，扩展性上有一定的限制。

## 编译器结构：LLVM 和 GCC 的区别

在编译一个程序时，一般会分为三个步骤：

1. **前端**：将编程语言转化为中间代码。可以理解为翻译的第一步，比如把你写的高级代码翻译成更抽象的“通用代码”。
2. **中端**：对代码进行各种优化。比如对循环进行优化，使程序运行得更快。
3. **后端**：将优化后的代码转化为目标机器能理解的代码（比如 CPU 指令）。

**LLVM** 使用一种叫做 **LLVM IR** 的中间代码，这种表示很灵活，可以支持多种编程语言和架构。

而 **GCC** 则使用两种中间表示：**GIMPLE** 和 **RTL**。GIMPLE 更像高级的中间表示，而 RTL 更接近底层硬件。举个例子，如果把编译过程比作修房子，GIMPLE 是房子的设计图，而 RTL 更像具体的施工细节。

### 实际应用举例

假如你要编译一个复杂的游戏程序，LLVM 会用 LLVM IR 让这个程序可以运行在不同的设备上，比如手机、PC、甚至游戏机。而 GCC 使用 GIMPLE 和 RTL 的分层表示，能够对特定硬件进行更加精细的优化，比如对于一些老旧硬件的支持，GCC 可能更有优势。

## 使用 GCC 和 LLVM 编译 Rust

### 使用 LLVM 编译 Rust

默认情况下，**Rust 使用 LLVM 来编译**。这意味着当你用 `rustc` 编译 Rust 代码时，LLVM 会帮你把代码转换成高效的机器代码。

例如，编译一个简单的 Rust 文件：

```bash
rustc example.rs
```

这样你就得到了一个可以运行的程序，而 LLVM 在背后帮助优化和生成了机器代码。

### 使用 GCC 编译 Rust

如果你想用 **GCC** 来编译 Rust，需要用到 `gccrs`。`gccrs` 目前还在开发中，但可以让 GCC 支持 Rust 编译。

假如你想试试 `gccrs`，可以按以下步骤操作：

1. 安装依赖：

   ```bash
   sudo apt install build-essential libgmp3-dev libmpfr-dev libmpc-dev flex bison autogen gcc-multilib dejagnu
   ```

2. 克隆 `gccrs` 项目：

   ```bash
   git clone https://github.com/Rust-GCC/gccrs
   ```

3. 构建工具链：

   ```bash
   mkdir gccrs-build
   cd gccrs-build
   ../gccrs/configure --prefix=$HOME/gccrs-install --disable-bootstrap --enable-multilib --enable-languages=rust
   make
   ```

构建完成后，你就可以用 GCC 来编译 Rust 代码了，不过目前的功能支持还比较有限。

### 实际应用举例

比如你在一台很老的电脑上想运行一个简单的 Rust 程序，用 LLVM 编译可能会遇到硬件不兼容的问题。这时候，GCC 的老旧硬件支持就派上用场了，因为 GCC 专门针对这些旧硬件做了很多年优化。

## 未来展望

LLVM 和 GCC 这两个编译器各有千秋。LLVM 更加灵活，特别适合那些需要在各种设备上运行的程序。而 GCC 历史悠久，支持很多传统的硬件，这些都是 LLVM 难以做到的。

Rust 社区的 `gccrs` 项目和 `rustc_codegen_gcc` 项目正在努力让 Rust 也可以用 GCC 编译。这会为 Rust 提供更多的选择和更强的兼容性，特别是在一些老旧设备上。

对于 Rust 社区来说，这是一件好事，因为它让 Rust 能够在更多平台上使用，这对于 Rust 语言的普及是很有帮助的。就像你选择不同的工具来完成工作一样，拥有不同的编译器选项，可以让开发者根据不同需求选择最合适的工具。

## 总结

- **LLVM**：模块化编译器，支持多种编程语言和架构，是 Rust 的默认编译器后端。
- **GCC**：老牌编译器集合，支持多种语言，稳定可靠，对传统硬件支持更好。
- **gccrs**：正在开发中的 Rust 前端，基于 GCC，可以为 Rust 提供更多的编译选择。

Rust 的编译器演变，代表了编程语言和编译器技术的不断进步。通过 LLVM 和 GCC 的合作，Rust 社区可以利用两者的优势，打造更高效和多样化的编译体验。
