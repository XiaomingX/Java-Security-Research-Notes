**网络犯罪分子利用流行游戏引擎Godot传播跨平台恶意软件**

一种流行的开源游戏引擎Godot Engine，最近被网络犯罪分子利用，作为一项名为GodLoader的恶意软件传播工具，自2024年6月以来，已感染超过17,000台系统。

Check Point公司在周三发布的分析报告中表示：“网络犯罪分子利用Godot引擎执行精心设计的GDScript代码，触发恶意命令并传播恶意软件。”该技术几乎能躲避所有的病毒扫描引擎，包括VirusTotal上的大多数防病毒软件。

网络犯罪分子不断寻找新的工具和技术，以便在避免被安全防护系统检测的同时传播恶意软件，而安全防护人员也在不断强化防线。

Godot Engine是一款支持跨平台的游戏开发引擎，允许用户设计2D和3D游戏，支持Windows、macOS、Linux、Android、iOS、PlayStation、Xbox、Nintendo Switch和网页等平台。它的多平台支持使其成为攻击者的理想工具，能够在更广泛的设备上进行攻击，从而扩大了攻击面。

**跨平台恶意软件的扩散**

Check Point的安全研究组负责人Eli Smadja表示：“Godot引擎的灵活性使其成为网络犯罪分子的目标，利用对开源平台的信任，像GodLoader这样的跨平台恶意软件得以迅速传播。”他补充道：“对于120万Godot开发游戏的用户来说，影响深远——不仅仅是他们设备的安全问题，甚至可能危及整个游戏生态系统的完整性。”

此次攻击的特别之处在于，黑客利用“Stargazers Ghost Network”——这是一个由约200个GitHub代码库和超过225个虚假账户组成的恶意分发网络。Check Point表示：“这些虚假账户通过在GitHub上标记恶意代码库，使其看起来像是合法和安全的。”这些代码库分四个波次发布，主要攻击目标为开发者、玩家和普通用户。

**攻击方式**

这些攻击发生在2024年9月12日、9月14日、9月29日和10月3日，通过Godot引擎的可执行文件（.PCK文件）传播GodLoader恶意软件。这个恶意加载程序会下载并执行最终的恶意负载，如RedLine Stealer（信息窃取器）和XMRig（加密货币挖矿病毒）。

此外，GodLoader还具备避开沙箱和虚拟环境分析的功能，并将整个C:\驱动器添加到Microsoft Defender防病毒的排除列表，以避免被检测到。

虽然目前的攻击主要针对Windows系统，但Check Point公司表示，这些恶意软件可以轻松修改以感染macOS和Linux系统。

**进一步升级的可能性**

目前，这些攻击通过定制的Godot引擎可执行文件来传播恶意软件，但攻击者还可以在获得解密密钥后，篡改合法的Godot开发游戏文件（.PCK文件）进行恶意传播。为了防范这种攻击，可以采用非对称加密算法（即公钥加密），通过公钥和私钥对数据进行加密和解密。

**安全建议**

Godot安全团队表示，Godot引擎本质上是一种编程系统，类似于Python和Ruby运行时，建议用户确保下载的可执行文件由可信方签名，避免使用破解软件。团队强调：“任何编程语言都可能被用于编写恶意程序，我们认为Godot并不比其他语言更容易被滥用。”

此次恶意软件攻击再次提醒我们，网络犯罪分子经常利用合法的服务和平台来规避安全机制，用户应该仅从可信的来源下载软件。

Check Point公司总结道：“攻击者利用Godot的脚本功能创建定制的恶意加载程序，能够避开许多传统安全解决方案的检测。由于Godot架构支持跨平台传递恶意负载，攻击者能够轻松地将恶意代码传播到Windows、Linux和macOS系统，甚至可能还会涉及Android平台。”

这种高效、低调的攻击方法使得恶意软件的传播速度异常快，且跨平台的特性大大增强了恶意软件的适应性和影响力，使得攻击者能够通过这种方式更有效地攻击多个操作系统，扩大攻击的范围和效果。
