# GHSL-2021-118: Zulip中的ReDoS（正则表达式拒绝服务） - CVE-2021-41115

Zulip软件中存在一个正则表达式漏洞，属于ReDoS（正则表达式拒绝服务）类型。

## 什么是ReDoS？

- **ReDoS**：这是一种攻击，利用不高效的正则表达式，导致在处理特殊输入时消耗过多时间，从而使服务变得不可用。

## 漏洞背景

- **发现方式**：该漏洞是通过CodeQL查询工具识别不高效的正则表达式发现的。
- **受影响的版本**：Zulip 4.5 版本

## 漏洞详情

- 存在两个不同的正则表达式容易受到攻击（具体定义在文中）。

## 重现漏洞步骤

1. **在Zulip组织中重现**：
   - 按照文中提供的步骤在真实的Zulip组织中测试。

2. **单独文件测试**：
   - 将易受攻击的正则表达式复制并粘贴到一个新文件中（步骤详见文中）。

## 风险说明

- 如果利用这个漏洞，可能会导致服务拒绝，影响用户使用。
