# GHSL-2020-295: ReDoS (正则表达式拒绝服务) 在 is.js 中 - CVE-2020-26302

## 概述
该项目中的一个或多个正则表达式存在 ReDoS 漏洞。ReDoS 是一种安全漏洞，可能导致由于不良构造的正则表达式在处理特定输入时性能极其差。

## 正则表达式拒绝服务 (ReDoS)
- **ReDoS**：当正则表达式效率低下或者结构复杂时，可能会导致在处理特定字符串时运行时间呈指数级增长（O(2^n)）。
- 这种问题通常源于正则表达式中的模糊性或重叠。

## 示例与修复方式
### 示例1: HTML 注释
- **漏洞正则**：匹配 HTML 注释的开始、任意字符和结束。
- **问题**：区分空格的方式过于灵活，导致在输入为 `<!--    ` 时，正则表达式程序会尝试多种匹配方式，耗时过长。
- **修复**：通过减少模糊性，确保正则表达式没有重叠。

### 示例2: snake_case 标识符
- **漏洞正则**：匹配以字母开头的snake_case 标识符。
- **问题**：正则结构中重复部分引入了多个匹配方式。
- **修复**：减少模糊性，确保内部的重复结构不再引起冲突。

## 实际案例
is.js 使用了一个从其他地方复制的正则表达式来验证 URL。如果输入恶意字符串，可能导致正则表达式“无限循环”。这个漏洞是通过 CodeQL 查询发现的。

## 影响
此问题可能导致拒绝服务攻击。
