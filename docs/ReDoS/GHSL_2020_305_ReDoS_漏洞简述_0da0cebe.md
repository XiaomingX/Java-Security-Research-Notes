# GHSL-2020-305: ReDoS 漏洞简述

## 概述
该项目中存在一个或多个正则表达式，容易受到 **ReDoS**（正则表达式拒绝服务）攻击。

### 什么是 ReDoS？
- **ReDoS** 是一种漏洞，发生在构造不当或效率低下的正则表达式上。
- 如果输入字符串经过精心构造，攻击者可以使正则表达式的执行时间变得极其缓慢，达到 **O(2^n)** 的运行时间。

## 漏洞原因
- ReDoS 通常由正则表达式中某些部分的模糊性或重叠性引起。
- 如果用户可以控制输入（如输入验证库），这些慢效能的正则表达式就可能导致安全问题。
- 修复 ReDoS 的一般方法是消除正则表达式内部的模糊性和重叠性。

## 示例
### 漏洞示例
#### HTML 注释匹配
```regex
<!--.*?-->
```
- 这个正则表达式尝试匹配 HTML 注释。
- `.` 匹配任何字符（除了换行符），而 `\s` 匹配空白字符。
- 当输入是一个包含长空格的字符串时，正则表达式会尝试所有匹配方式，导致性能下降。

#### 修复方法
将正则表达式改为更具体的版本，消除模糊性。

### 另一个示例
#### snake_case 标识符匹配
```regex
(\w+_?)+
```
- 这个表达式尝试匹配以字母开头的字符串，允许不分组的重复。
- 输入以大量字母开头时，也会导致多种匹配方式。

#### 修复方法
将内层重复修改为必须以下划线结束的版本，减少模糊性。

## 复杂示例
- 有些正则表达式复杂得多，例如曾在 VS Code 中使用的正则表达式。
- 在这里，通过调整过于宽泛的部分（如 `\S`），使其更具体（比如 `[^\s\(\)]`），消除重叠问题。
