# GHSL-2023-121: SAML身份验证绕过漏洞（CVE-2023-48703）

## 概述
在RobotsAndPencils的go-saml库中发现了一个SAML身份验证绕过漏洞。这个问题可能导致使用go-saml进行SAML断言签名验证的应用程序出现身份验证绕过。

> **注意:** go-saml库已被标记为归档状态，不再维护。仍在使用该库的项目应考虑迁移到其他SAML库，或从项目中去除SAML支持。

## 漏洞细节
- **问题来源**: 在内部调用xmlsec1命令行工具以验证SAML断言的签名时，未明确指定受支持的公钥数据。这意味着攻击者可以自己签署SAML断言并在SAML令牌中嵌入所需的公钥（如RSA密钥）。
  
- **代码流**: 使用go-saml的应用程序调用`Validate`方法来验证SAML响应。该方法将调用`VerifyResponseSignature`，进一步调用xmlsec.go中的`verify`函数。这里使用的xmlsec1工具没有参数限制，这导致它会考虑XML文件中嵌入的公钥。

### 受影响的调用：
- `VerifyResponseSignature`
- `Validate`（在`authnresponse.go`中）
- `VerifyRequestSignature`
- `Validate`（在`authnrequest.go`中）

## 暂时解决方案
如果项目仍然使用go-saml库，可以通过以下方式进行临时修复：
- 分叉go-saml项目，向`verify`函数中添加命令行参数`--enabled-key-data`，并指定例如`x509`或`raw-x509-cert`作为值，调用xmlsec1工具。

> **注意:** 此临时解决方案必须经过仔细测试。

## 版本信息
- 2024年1月发布的xmlsec1版本1.3.3默认禁用了KeyValue和DEREncodedKeyValue XML节点。然而，大多数Linux发行版使用的是旧版本的xmlsec1，因此建议在任何情况下都指定`enabled-key-data`。

## 建议
强烈推荐从go-saml库迁移到其他库，因为该库已不再维护。
