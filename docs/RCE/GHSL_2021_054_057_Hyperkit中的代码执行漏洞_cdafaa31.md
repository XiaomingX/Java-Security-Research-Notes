# GHSL-2021-054_057: Hyperkit中的代码执行漏洞

此文介绍了**hyperkit**中的多个漏洞（CVE-2021-32843到CVE-2021-32846），这些漏洞允许恶意虚拟机（guest）利用指定的驱动程序，使得恶意代码在宿主机（host）上执行，从而导致服务拒绝（Denial of Service, DoS）。

## 技术基础

### Hyperkit

- **Hyperkit**是一个用于虚拟化的工具，主要用于运行虚拟机（如Docker中的虚拟环境）。
- 设备通过结构体 `virtio_consts` 实现，里面包括了处理程序和组件。

### Virtio 设备

- Virtio是一种支持虚拟化设备的标准。每个设备可以实现一些操作，包括读取和写入配置。
- 设备可能并不实现所有可用的操作，因此在调用这些操作之前，必须检查指针是否为**null**。

### vtrnd设备

- **vtrnd**是一个能够给虚拟机提供随机数的Virtio设备。
- 当尝试读取配置时，如未检查指针是否为null，可能导致宿主机崩溃。

## 漏洞概述

### 漏洞示例

1. **vc_cfgread**:
   - 如果调用未检查null，可能会使宿主机崩溃。
  
2. **vc_cfgwrite**:
   - 类似`vc_cfgread`，在调用未检查情况下，宿主机会崩溃。

3. **vq_getchain**:
   - 该函数用于读取虚拟机的随机数。如果处理错误未被检查，则会发生内存未初始化的读操作，可能导致宿主机崩溃和内存损坏。

4. **pci_vtsock_proc_tx**:
   - 返回值检查不足，可能导致进一步的内存损坏。

### 漏洞后果

- 恶意虚拟机可以崩溃宿主机，导致服务拒绝。
- 在某些情况下，可能发生内存损坏。

## 漏洞验证

- 为验证这些漏洞，可以使用简单的多重引导内核作为证明概念（PoC），需要在启动时使能vtrnd设备。

### 编译说明

- 将漏洞分析代码放入C文件，并添加文件`linker.ld`和`start.s`。
- 需要安装 `nasm` 和 `i686-elf-gcc` 进行编译。
