# GHSL-2020-228: YApi中的弱JSON Web Token (JWT) 签名秘密 - CVE-2021-27884

## 问题概述
YApi使用了不安全的随机数生成器来签名JWT，这可能导致安全隐患。

## 技术背景
- **JWT (JSON Web Token)**: 一种用于安全传输信息的令牌。
- **伪随机数生成器**: 用于生成看似随机的字符串，但并不安全。

## 问题细节
- **随机字符串生成**: YApi使用`Math.random()`函数生成密码盐，这个过程并不安全。
  - 生成的字符串大约10-12个字符长，仅包含数字和小写字母。
  - 生成的字符串是可逆的，攻击者可以通过算法还原出密码盐。

- **攻击方式**:
  - 攻击者可以快速创建多个用户账户，获取JWT。
  - 通过观察三个连续的JWT值，攻击者可以重建随机数生成器的内部状态。
  - 这一攻击虽然在单台机器上耗时较长，但借助云计算，攻击成本大约在8,000到24,000美元之间。

## 风险
- 攻击者能够获取和重构用于签名的密码盐，可能伪造其他用户的JWT。
- JWT的有效期为7天，不会限制攻击的窗口期。
