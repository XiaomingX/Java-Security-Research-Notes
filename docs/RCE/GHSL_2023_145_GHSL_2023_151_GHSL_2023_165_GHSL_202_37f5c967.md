# GHSL-2023-145_GHSL-2023-151/GHSL-2023-165_GHSL-2023-172: 关于stb_image和stb_vorbis库的内存访问漏洞

## 概述
在`stb_image.h`和`stb_vorbis`库中发现了多个不同严重程度的内存访问漏洞，这些问题可能导致信息泄漏、服务拒绝、或代码执行。

### 受影响的库版本
- `stb_image` - v2.28
- `stb_vorbis` - v1.22

## 漏洞描述

### 1. 内存泄漏
- **问题**: 当使用`stbi_set_flip_vertically_on_load`设置为`TRUE`且`req_comp`与实际像素分量数量不匹配时，可能导致越界的内存读取。
- **原因**: 内部计算的行字节数与实际图像尺寸不符，导致读取超出分配的内存区域。

### 2. double-free漏洞
- **问题**: `stbi__load_gif_main`函数可能试图释放未分配的内存。
- **场景**: 在处理图像层时，当层的步幅为零时，可能触发重复释放的尝试，这在多线程环境中尤为危险。

### 3. 空指针访问
- **问题**: 如果某些函数返回了空指针，代码会继续执行并试图访问该指针。
- **后果**: 这可能导致程序崩溃（服务拒绝）。

### 4. 内存泄漏与重复释放
- **问题**: `stbi__load_gif_main`函数在失败时可能未能重置内存指针，造成隐蔽的内存泄漏。
- **后果**: 调用者可能在未处理失败情况下继续释放内存，导致错误的内存管理（重复释放或内存泄漏）。

### 5. 越界写入
- **问题**: 某些情况下，写入的位置超出了分配的内存区域。
- **原因**: 由于整数溢出或读取的长度不正确，可能导致向无效地址写入数据。
- **后果**: 这可能被攻击者利用来执行任意代码。

### 6. 内存分配失败
- **问题**: 如果在内存分配时遇到失败，部分指针未能被正确初始化或重置。
- **后果**: 程序可能尝试解引用未初始化的指针，可能导致崩溃。

## 结论
上述漏洞严重影响了`stb_image`和`stb_vorbis`库的安全性，可能被恶意用户利用。建议用户及时更新到最新版本以防止潜在的安全风险。
