# GHSL-2020-131: Mongo-express中的远程代码执行漏洞 - CVE-2020-24391

## 背景

mongo-express是一个MongoDB的Web管理工具。它使用`safer-eval`来验证用户提交的JavaScript代码，但这个验证方法存在漏洞，允许攻击者在Node.js服务器上下文中执行任意代码（RCE）。

### 关键点

- **受影响版本**：mongo-express v0.54.0
- **漏洞类型**：远程代码执行（RCE）

## 漏洞细节

当用户提交JavaScript代码时，mongo-express会通过`safer-eval`进行验证。然而，由于该工具的沙箱机制可以被轻易绕过，攻击者能够执行未授权的命令。

### 漏洞利用

攻击者可以通过以下方式利用此漏洞：

1. **需要用户认证**：必须有一个已登录的用户。
2. **无意识的POST请求**：通过特制的负载，攻击者可以自动向`/checkValid`端点发送请求，并执行用户提交的代码。

### 漏洞验证步骤

1. 创建一个名为`index.html`的文件，内容包含特制的代码。
2. 在相同目录启动一个Web服务器。
3. 在浏览器中访问 `http://localhost/`，确保目标mongo-express服务器中有一个已登录用户，并且会话是在同一浏览器中启动的。
4. 检查mongo-express的控制台，可以看到请求已成功发送，并且执行了系统命令。
