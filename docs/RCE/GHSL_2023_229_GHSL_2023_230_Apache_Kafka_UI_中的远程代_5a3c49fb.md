# GHSL-2023-229_GHSL-2023-230: Apache Kafka UI 中的远程代码执行漏洞

## 概述
Apache Kafka UI 存在两个远程代码执行（RCE）漏洞，编号为 CVE-2023-52251 和 CVE-2024-32030。以下是这两个漏洞的详细介绍：

## 漏洞细节

1. **漏洞一：无沙箱的 Groovy 脚本执行**
   - Kafka UI 中的消息过滤组件允许用户使用 `GROOVY_SCRIPT` 类型的过滤器。
   - 攻击者可以利用这一点，在服务器上执行任意 Groovy 脚本，不受沙箱限制。

2. **漏洞二：通过 JMX 服务器的反序列化攻击**
   - Kafka UI 允许连接到 JMX 服务器，如果连接的是恶意服务器，攻击者可以利用不安全的反序列化机制远程执行代码。
   - Kafka UI 默认情况下没有启用身份验证，这使得漏洞更加危险。如果启用了身份验证，攻击者仍需有效账号才能利用漏洞。

## 技术背景
- **Groovy 脚本过滤器**
  - 该过滤器使得攻击者可以查看消息内容及其属性，并执行任意代码。
  - 漏洞方法位置：`com.provectus.kafka.ui.emitter.MessageFilters#groovyScriptFilter`

- **JMX（Java Management Extensions）**
  - 允许监控 Kafka 代理性能，通过 RMI（远程方法调用）协议进行。
  - 攻击者可以创建伪造的 RMI 监听器，返回恶意序列化对象，从而实现代码执行。

## 漏洞利用步骤
1. **设置 RMI 监听器**
   - 攻击者在自己的系统上设置伪造的 RMI 监听器，使其返回恶意对象。

2. **触发反序列化**
   - Kafka UI 连接到该恶意 RMI 监听器，触发恶意对象的反序列化。
   - 该过程可能记录错误日志，该行为正常。

3. **执行命令**
   - 如果在窝藏过程中，系统属性 `org.apache.commons.collections.enableUnsafeSerialization` 被设置为 true，接下来的反序列化将导致攻击者指定命令的执行。

## 结论
这些漏洞的有效利用可能导致即使在身份验证后也可进行远程代码执行。用户应小心使用 Kafka UI，尤其是在没有进行适当安全配置的情况下。
