# GHSL-2023-028: Jellyfin中的远程代码执行漏洞 - CVE-2023-48702

## 概述
在jellyfin服务器上，一位拥有管理员权限的用户可以通过`/System/MediaEncoder/Path`接口执行任意代码。

## 技术细节

- **接口与函数**: 
  - `/System/MediaEncoder/Path`接口能够执行任意文件，这是通过`ProcessStartInfo`和`ValidateVersion`函数实现的。
  
- **利用路径**:
  - 恶意管理员可以设置网络共享并提供一个指向网络共享上可执行文件的UNC路径，从而使jellyfin服务器在本地上下文中运行该可执行程序。

## 漏洞利用示例

1. **身份验证问题**:
   - 使用curl工具直接访问`/System/MediaEncoder/Path`时遇到了一些身份验证问题，因此建议使用web用户界面进行操作。

2. **上传恶意文件**:
   - `/Branding/Splashscreen`接口无法严格验证文件类型。它根据请求头中指定的内容类型创建文件扩展名，但未检查文件本身的类型。
   - 可以通过指定有效的内容类型（例如`image/png`）来上传可执行文件。

3. **结合漏洞**:
   - 将上述两个漏洞结合，恶意用户可以利用`/System/MediaEncoder/Path`接口实现远程代码执行（RCE）。

## 风险
此漏洞可能导致在身份验证后进行远程代码执行。
