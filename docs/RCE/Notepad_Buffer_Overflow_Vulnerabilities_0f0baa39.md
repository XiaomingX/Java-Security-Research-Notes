# Notepad++ Buffer Overflow Vulnerabilities

## 概述
Notepad++ 存在多个缓冲区溢出漏洞，影响版本为 **v8.5.2**。这些漏洞可能导致任意代码执行。

## 技术细节

### 1. CVE-2023-40031
- **问题：** `Utf8_16_Read::convert` 函数在转换 UTF16 到 UTF8 时，计算的新缓冲区大小错误。
- **原因：** 输入长度为奇数时，计算不正确。例如，当输入长度为 9，计算新大小为 14，但偏移量导致读取越界。
- **复现步骤：**
  1. 编译 Notepad++ 的 ASAN 版本。
  2. 打开特制文件以触发越界访问。

### 2. CVE-2023-40036
- **问题：** 由于索引越界，读取超过全局分配的对象缓冲区。
- **原因：** 输入文件内容导致数组索引超出 `mCharToFreqOrder` 的大小。
- **复现步骤：**
  1. 编译 ASAN 版本或设置断点。
  2. 打开特制文件以触发越界访问。

### 3. CVE-2023-40164
- **问题：** 数组索引 `byteCls` 超出 `charLenTable` 的大小。
- **原因：** 输入文件内容设置的索引超出了有效范围。
- **复现步骤：**
  1. 编译 ASAN 版本或设置条件断点。
  2. 打开特制文件以触发越界访问。

### 4. CVE-2023-40166
- **问题：** 文件加载时超出数据缓冲区的末尾读取。
- **原因：** 无法正确检查读取的字节长度。
- **复现步骤：**
  1. 为 `notepad++.exe` 启用页面堆。
  2. 创建特制文件并尝试打开以触发问题。
