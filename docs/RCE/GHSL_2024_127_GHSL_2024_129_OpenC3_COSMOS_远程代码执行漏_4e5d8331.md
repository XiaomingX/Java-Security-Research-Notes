# GHSL-2024-127_GHSL-2024-129: OpenC3 COSMOS 远程代码执行漏洞报告

## 概述

在 OpenC3 COSMOS （一个用于控制卫星和测试设备的 web 应用程序）中发现了几个漏洞，这些漏洞可能导致远程代码执行（RCE）和信息泄露。主要缺陷来自跨站脚本攻击（XSS）。

## 主要漏洞

### 1. 路径遍历漏洞

- **版本**: OpenC3 COSMOS v5.16.2
- **描述**: 经过身份验证的用户可以利用 `ScreensController#show` 方法中的路径遍历漏洞下载服务器上的任意 `.txt` 文件。
- **原理**:
  - 用户提供的参数 `target` 和 `screen` 被用于构造文件路径，这个路径最终被送入 `File.open` 进行打开。
  - 必须通过特定的 URL 参数构造文件路径，例如：要访问 `/openc3/python/requirements.txt`，需提供合适的参数。
  
```bash
curl -X GET "http://<cosmos-host>/path?target=<your_target>&screen=<your_screen>"
```

### 2. 反射型跨站脚本（XSS）漏洞

- **描述**: 登录功能中存在 XSS 漏洞，攻击者可以通过特制的重定向参数注入恶意 JavaScript 代码。当用户登录后，这段代码可以执行。
- **原理**:
  - 攻击者诱导用户访问特定链接，当用户登录时，输入的密码会被恶意 JavaScript 代码捕获并发送给攻击者。
  
### 3. 不加密的密码存储

- **描述**: OpenC3 COSMOS 在浏览器的 LocalStorage 中以明文形式存储用户密码，这使得密码容易被窃取。
- **原理**:
  - 攻击者可利用 XSS 漏洞获取 LocalStorage 中的密码，并将其发送到控制的远程服务器。

## 结论

以上漏洞均可能导致信息泄露及远程代码执行。建议尽快更新至安全版本并加强安全措施。
