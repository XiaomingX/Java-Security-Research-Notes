# GHSL-2023-238_GHSL-2023-244: Ngrinder中的安全漏洞概述

Ngrinder是由Naver开发的一款Web应用程序，发现了多个安全漏洞，包括 **未认证的远程代码执行（RCE）** 和 **不当的访问控制**。以下是这些漏洞的重要技术知识点。

## 漏洞总结

### 1. 远程代码执行（RCE）漏洞
- 两个API端点允许未认证的攻击者执行任意命令。
- **/monitor/api/state?ip={ip}**：允许攻击者指定IP地址，让Ngrinder连接其控制的JMX/RMI服务器。这可以让攻击者运行任意命令。
  - 攻击者通过`ip`参数传递一个恶意的RMI地址，执行`touch /tmp/pwnedGroovy.txt`命令。
  
- **/script/api/github/validate**：通过未安全处理的YAML反序列化，允许攻击者执行任意代码。
  - 传递的YAML数据被不安全地传递到一个反序列化点，使攻击者可以执行命令。

### 2. 远程拒绝服务（DoS）
- **/check/healthcheck_slow**：此端点接受来自用户的任意延迟请求，攻击者可以发送大量高延迟的请求来占用服务器线程，导致服务不可用。

### 3. 不当访问控制导致的风险
- 攻击者可以未认证地创建或更新Webhook配置，可能导致信息泄露或有限的服务器端请求伪造（SSRF）。
- 攻击者能够查看Webhook请求的执行结果，包括HTTP头和消息体，造成信息泄露。

### 4. 不需要认证的报告查看
- 多个性能报告的端点不需要认证，任何人都可以查看报告，导致信息泄露。

## 漏洞的影响
- 上述漏洞可能导致：
  - 未认证的远程代码执行（RCE）
  - 远程服务中断（DoS）
  - 信息泄露
  - 非授权的配置修改

## 安全建议
- 建议尽快更新Ngrinder到最新版本以修复这些漏洞。
- 实施适当的访问控制和输入验证机制。
