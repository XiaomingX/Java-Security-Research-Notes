# GHSL-2023-190: Frigate中的多个漏洞 - CVE-2023-45672, CVE-2023-45671, CVE-2023-45670

## 概述
Frigate软件存在几个安全漏洞，允许攻击者执行恶意代码或进行XSS攻击。以下是详细说明：

## 漏洞详情

### 1. 不当的日志访问控制
- **问题**: 在`logs/<service>`接口中，用户可以请求特定服务的日志（如nginx、go2rtc、frigate）。如果用户指定的服务不存在，系统应该返回错误，但实际上却返回未处理的HTML内容。
- **影响**: 攻击者可以注入JavaScript payload，造成跨站脚本(XXS)攻击。

### 2. 代码执行漏洞
- **路径**: 通过`frigate/http.py`接收用户输入，接下来在`frigate/config.py`解析输入。
- **问题**: 输入通过`yaml.loader.Loader`处理，如果提供恶意payload，将直接执行。
- **影响**: 可能导致未经认证的远程代码执行（RCE）。

### 3. 反射型XSS漏洞
- **问题**: `recording_clip`请求处理器根据请求的摄像头名称返回字符串，由于未转义，可能引发反射型XSS。
- **举例**: 通过请求一个不存在的摄像头，从而制造返回未处理的输入，允许JavaScript代码执行。

### 4. 缺少CSRF保护
- **问题**: `config_save`和`config_set`请求处理器没有CSRF保护，可以接收任意来源的请求来更改配置。
- **影响**: 允许未授权更新Frigate服务器的配置，可能导致拒绝服务和数据泄露。

### 5. 用户控制的错误信息
- **问题**: 返回的错误信息内容类型为text/html，且参数完全由用户控制，未经过滤。
- **影响**: 攻击者可以构造恶意URL，并注入XSS payload。

## 结论
这些漏洞可能导致严重的安全问题，包括代码执行和信息泄露。GHSL团队成员@maclarel和@jorgectf发现并报告了这些问题。
