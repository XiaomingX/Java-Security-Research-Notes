# GHSL-2024-100 to GHSL-2024-108: Streamlit-geospatial 中的漏洞

### 概述
Streamlit-geospatial 项目存在多项安全漏洞，包括远程代码执行 (RCE) 和盲服务器端请求伪造 (SSRF)。

### 关键技术知识点

1. **远程代码执行 (RCE)**:
   - 当用户输入的数据被错误地用于执行代码时，就会出现 RCE 漏洞。在以下情况下，用户输入会通过 `eval()` 函数执行，导致潜在的远程代码执行：
     - `pages/1_📷_Timelapse.py` 中的 `palette` 和 `vis_params` 变量，在多行代码（如 380, 395, 430, 493, 1254, 1345, 115, 126, 80, 86）中接收用户输入并被执行。

2. **盲服务器端请求伪造 (SSRF)**:
   - SSRF 漏洞允许攻击者发送请求到内部或外部的任意地址，而不需要直接与目标交互。这主要发生在以下几个函数中：
     - `pages/1_📷_Timelapse.py` 中，用户输入的 `url` 参数被传递到 `get_layers` 函数，导致发送请求到任意目的地。
     - `8_🏜️_Raster_Data_Visualization.py` 中，用户输入的 `url` 参数被传递给 `gpd.read_file` 方法，同样引发 SSRF。

### 风险
- RCE 漏洞可能导致攻击者在受害服务器上执行任意代码。
- SSRF 漏洞可以被用来探测其他系统中的漏洞，尤其是在攻击者可以访问的内部网络。
