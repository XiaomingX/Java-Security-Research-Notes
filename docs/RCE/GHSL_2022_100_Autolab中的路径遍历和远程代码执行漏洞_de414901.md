# GHSL-2022-100: Autolab中的路径遍历和远程代码执行漏洞

在Autolab中发现了两个漏洞，分别是CVE-2022-41955和CVE-2022-41956。以下是详细介绍：

## Autolab概述

- **版本**：v2.9.0
- **功能**：允许学生通过“远程提交”功能提交作业。提交的文件来自运行Autolab实例可以访问的路径，通常是在共享驱动器的某个目录内。

## 漏洞描述

### 1. 路径遍历漏洞（CVE-2022-41955）

- **问题**：攻击者可以利用“远程提交”功能提交其他用户的作业。提交时，攻击者可以构造一个URL，包含被攻击者提交目录以外的路径。
- **结果**：攻击者可以查看或下载敏感文件（如 `/etc/passwd`），这可能导致信息泄露。
  
**利用示例**：
  
- 攻击者可以发送GET请求，格式如下：
  ```
  http://<autolab-url>/submit?submit=<path_to_sensitive_file>
  ```

### 2. 远程代码执行漏洞（CVE-2022-41956）

- **问题**：Autolab的剽窃检查功能允许教师在某些条件下执行命令。构造的参数没有经过验证，攻击者可以插入恶意命令。例如，在“Max Lines”输入框中输入一个shell命令。
- **结果**：攻击者可以在Autolab服务器上执行任意命令。

**利用示例**：

- 攻击者在选择作业后，可以输入如下内容：
  ```
  | touch /tmp/pwned.txt |
  ```
- 这会创建一个名为 `/tmp/pwned.txt` 的文件。

## 如何防护

- 确保对用户输入的参数进行严格的验证和消毒。
- 检查系统的访问权限，限制哪些用户可以执行敏感操作。
