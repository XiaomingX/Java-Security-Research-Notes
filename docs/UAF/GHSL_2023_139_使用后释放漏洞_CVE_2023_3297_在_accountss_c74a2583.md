# GHSL-2023-139: 使用后释放漏洞 (CVE-2023-3297) 在 accountsservice 中

## 概述
一个低权限的本地攻击者可以通过向 `accounts-daemon` 进程发送 D-Bus 消息，利用 `accountsservice` 中存在的使用后释放 (Use After Free, UAF) 漏洞。

## 受影响的版本
- `accountsservice` 版本: 22.08.8-1ubuntu7
- 漏洞在 Ubuntu 23.04 和 22.04 LTS 中均存在，但在 23.04 上更容易观察到。

## 问题描述
当 `D-Bus` 服务器接收到方法调用时， 应该只发送 `METHOD_RETURN` 或 `ERROR` 消息，而不能同时发送。`accountsservice` 并未正确遵守这一协议，在多个地方存在这一问题。

### 示例
在 `user_change_language_authorized_cb` 函数中：
- 如果 `user_HOME_available` 返回错误，则会调用 `throw_error` 发送 `ERROR` 消息，同时又错误地发送了一个 `METHOD_RETURN` 消息。
- 这会导致使用后释放的问题，因为 `throw_error` 和 `accounts_user_complete_set_language` 都减少了 `context` 的引用计数。
- 简而言之，`context` 被 `throw_error` 释放后，`accounts_user_complete_set_language` 将访问已释放的内存，造成 UAF。

## 攻击方式
攻击者可以通过使 `user_HOME_available` 失败来触发该漏洞。例如，攻击者可以删除其主目录中的所有文件。还有其他方法可以触发 `throw_error`，比如使用包含无效字符的字符串。

### 影响
- 在 Ubuntu 23.04 上，执行上述命令会导致 `accounts-daemon` 崩溃，出现 SIGSEGV 错误。
- 在 Ubuntu 22.04 LTS 上则没有明显的危害，差异原因在于 GLib 内存分配的变化：新版本使用了系统分配器，使得释放内存的方式导致 UAF 明显引发崩溃。

## 安全影响
虽然利用此漏洞可能比较困难，但它可能使低权限的本地攻击者获取 root 权限。
